Inventory enhancements and changes

Our partners have spoken and as always we have listened. Service items that allow customer to sell services by e.g. hour or work is now part of LS One. It complements the already support for retail items and variant headers that forms the basis anything being sold.

This page includes all changes that were done to the inventory including smaller improvements and bug fixes that are specifically for the inventory functionality

Service items

Functionality for service items has now been added to LS One. Service items are very useful when customers need to sell items such as modifications, repairs or any service that does not involve actual stock being sold.

When creating a new item in the Site Manager a new option has been added where the item can be created as a service item. A service item is handled the same as any item with in the LS One POS with the exception that it will not generate any inventory movements. On-hand inventory for these items will always be zero. Service items can be added to customer orders in the LS One POS as any other items can be.

Within the inventory functionality in the Site Manager these items will not be visible at any time as they cannot create or maintain inventory movements.

An existing retail item can be converted into a service item through a special dialog on the item card. If the retail item already has on hand inventory then the user changing the item needs a specific permission, Manage item types, to be able to change it. If the item on the other hand has no on hand inventory a normal Manage retail items permission is enough. When changing an item from retail item to a service item the functionality will create inventory adjustments to clear out (zero out) the inventory for the item which is the reason for the special permission.

If an item that is changed to be a service item has already been added to a purchase order or stock counting journal the functionality will not stop the user from changing the item to a service item but the inventory will display the item with a strike through text and not include the item when the purchase order, goods receiving document and etc. are posted.

When opening up the item card in the Site Manager for a service item the Vendor tab, Replenishment and Inventory tabs are disabled.

Deleted items and service items

If an item that has already been added to a purchase order, goods receiving document, inventory adjustment, purchase order worksheet or any of the other inventory documents is either deleted or changed to a service item those items will be displayed with a strike through text and will then not be included in the posting procedure when the document is posted or a goods receiving document is created from a purchase order for example.

User experience changes

All lists where previously the coloring of the line was used indicate a status of the inventory documents have now been changed to have an icon in the at the beginning of every line to indicate the status. An example of these icons would be a dark circle with a check mark to indicate posted documents and then circles with various other colors to indicate other statuses.

Purchase orders

Gray circle: Open purchase order

Dark circle with a check mark: Posted purchase order

Goods receiving document

Gray circle: Open document

Yellow circle: Partially received

Dark circle with a check mark: Posted goods receiving document

The indication of if the received quantity is the same, less then or greater than ordered have also been changed to be text color rather then cell color.

Stock counting journal

Gray circle: Open document

Yellow circle: Partially posted

Dark circle with a check mark: Closed

The indication of if the counted quantity is the same, less then or greater than the on hand inventory has also been changed to be text color rather then cell color.

Other improvements

If a vendor is deleted (archived) it is not visible anymore in any combo boxes or lists in the Site Manager

When creating a purchase order line for a vendor item that has a default purchase price the purchaser order line dialog will retrieve the vendors purchase price

When doing a stock counting in another unit than inventory unit the difference of the current on-hand inventory and the counted stock was not calculated correctly

When 'return item' was used to return an item the inventory was decreased instead of increased. This has now been fixed so the return item and return transaction operations behave the same.

A data scroller has been added to the purchase order worksheets to limit the number of items that are retrieved through the site service into the worksheet.

When adding items to a transfer request or order and a new unit is selected the Site Manager will now prompt the user for a unit conversion.

Inventory unit is now selected by default when adding item to inventory transfer requests and orders.

When transfers orders had more "sent" quantity then "received" quantity the information about the transfer was still being displayed in Inventory in transit. Now once the transfer has been marked as Recieved it will not be displayed in this view anymore. This change will affect transfers that were created in previous releases of LS One POS as this was a database issue not user interface issue.

When importing an excel stock counting sheet the store ID column for the stock counting journal header is now a mandatory column

When deleting units now conversion rules are checked to make sure that no rule is using the unit when deleting it. The system also checks if the unit is being used in any inventory process

Now inventory levels for an item for all stores can be viewed with Site Manager set on a specific store not only Head office. If the Site Manager is configured to be on a specific store the on hand information for that store is displayed in bold

When a stock counting journal is deleted now all the stock counting lines are also deleted (not just the header as before).

If an inventory template has an open purchase worksheet on it then the template cannot be deleted. Please note that a worksheet is considered open until it is cleared by "Posting" it.

When editing vendors for a specific item the OK button was not being enabled and disabled properly.

When creating new Stock reservation lines and Inventory adjustment lines the unit description is now displayed in the list view.

Edit button on purchase order worksheets list is now disabled if the list is empty and when no worksheet is selected

When importing an excel stock counting sheet the store ID column for the stock counting journal header is now a mandatory column
