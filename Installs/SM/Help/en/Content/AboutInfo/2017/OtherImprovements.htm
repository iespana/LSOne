Other improvements

Here various new and improve functionalities are mentioned and explained.

Serial number functionality

The LS One POS has had functionality for serial numbers for a long time but the functionality to view and import serial numbers into the Site Manager has not been available before now. With the variant framework changes that were done in the last release the serial number functionality had to be disconnected.

In this release Serial numbers can be added to items in the Site Manager, they can be manually entered or imported through either an Excel file or a CSV file. The item needs to be configured to ask for serial numbers (see Key in serial numbers) and if serial numbers are imported into the system then the user will get a list of serial numbers to choose from, but the user can always manually enter a serial number if needed. If no serial numbers are added/imported the POS will ask for a manual input of a serial number. The serial number functionality requires a Site service connection as the serial numbers are updated as soon as they are selected (as reserved) and then once the item is paid for information about receipt ID, date and time as added to the serial number.

Please see our help topic on this issue for further information.

Please note that the serial numbers are only handled in the LS One POS, the functionality to handle serial numbers in the inventory has not been implemented.

Additional barcode functionality

Barcodes assigned to gift cards and credit memos can now be scanned directly into the LS One POS and payment operations for these types of payments will be activated (Pay gift card and Pay credit memo). For further information on how to configure barcodes for gift cards and credit memos please see our help topics on the issue.

A new section for gift cards was created in the Receipt designer with a variable so that the gift card or credit memo number can be printed as a barcode.

Periodic discounts that are configured to be triggered manually can now be activated by a barcode as well as by a button. Barcode setup needs to be done for the POS to recognize the barcode as a discount barcode and the barcode needs to be added to the periodic discount itself as well.

Embed terminal ID in receipt ID

A new option that embeds a terminal ID to a number sequence has been added to the number sequence view. By default only the receipt ID is configured to have this active in a new empty database. When updating an existing database the receipt ID number sequence is not updated to embed the terminal ID. To update an existing customer to have this option the change needs to be done manually through the database table itself.

Payment limitations

Configurations for the payment limitations have been reviewed and updated. Limitations can be created on specific items, retail group, special group, retail department and everything. The limitations can be set to be inclusive or exclusive. To activate these limitations they need to be attached to the payment method on the Allowed payment method tab on the Store card. In the release the pay operations for cash, currency, customer account and check have been implemented to respect the payment limitations. But other payment methods (gift cards, loyalty, credit memo, cards) needs to be implemented in a customization.

System reports

Three of the system reports have been updated and need to be re-imported into the system. Those are Financial report, Reorder point by vendor and Stock level report by vendor. These reports are part of the report package on the partner portal and can be downloaded there.

Concluding the transaction

Before anything is done when concluding the transaction in the LS One POS we now check to see if there is anything that could possible require the site service and if so then we test the connection to the site service. If the test is not successful then a message is displayed saying what needs the site service connection and that the operation cannot continue

LS One POS improvements

Variant column in the Item search list in the POS is now not visible except when there are variant items in the list

Fixed an error that was displayed if a manual discounts was applied to an item that had a price of zero

Discount offer price for variant items is now calculated from variant item price not variant header price as before

An infocode that is added to a card type was not getting activated in when payment was done using that card type. This issue has now been fixed.

When returning a sale with a customer line discount the discount was not calculated again and included in the return.

X/Z report width can now be configured in the Functionality profile. The default value is 55 which is how it was before.

The sales person list in the POS does not display deleted users anymore

Operations Customer transactions, Customer transactions report and Customer balance report has been set as non-user operations and cannot be selected on a button

If a gift card is being refilled and the transaction is voided the gift card is not deleted anymore. But the refillable amount is voided.

Site Manager improvements

Now all reports can be viewed from the Tools > View report button in the ribbon. The ribbon us updated at run time so when a new report is imported the Site Manager needs to be restarted for the report to appear in the list of reports.

When creating a new item, if the retail group selected has a tax group on it, that tax group is set on the new item automatically.

When the store property on the terminal is changed then the terminal is now moved to the other store. Before the terminal was duplicated.

When adding a linked item to another item a variant header item cannot be selected.

In a few list views, in the right click menu, the "Add" option was named "New". This has been changed to "Add"

The paging in the Item discount groups view was not working properly and has been fixed.

A change has been done that does not allow a deleted item to be edited anymore. In order to edit a deleted item it has to be activated again.

Both item sales tax group view and customer sales tax view now have a "Add another" check box on the dialogs that add the tax codes, to allow the user to add more than one tax code at a time.

All fields are now visible in the Triggers tab on the View jobs card when the Site Manager is set to be in it's smallest size

When a customer is deleted, it is now in effect archived so that is can be activated again. Any reports that deal with customer information will still be able to display customer information even if they have been deleted. A customer also cannot be deleted if for example he has a negative balance, any customer orders outstanding and etc. Deleted customer can be seen by selecting the deleted option in the search bar in the customers view

The styles view has been updated considerably and all system styles have now been updated to have descriptions that can be translated. Also the system styles can be set back to "factory settings" so that an existing customer can get new updated system styles or change them back to how they were originally.

The number sequence view has been updated and a search bar has been added to the list view to make finding a specific number sequence easier.

The user can now sort by "Variant" name in the retail items list

When importing an item that has been previously deleted through the Excel import a message is now displayed explaining that the item has been deleted and cannot be imported again.

The virtual printer has been retired and to test receipts the hardware profile can be configured to print to a "Windows printer" and then select for example Snagit or any other image editor that can be used as a printer

An additional search option on the items view has been added where the list can be filtered by retail items or service items

Minor improvements that touch the inventory functionality can be viewed in the inventory section of the release notes
