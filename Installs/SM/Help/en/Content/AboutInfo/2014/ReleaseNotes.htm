2014 Release notes

This release includes various functionality changes as well as minor changes and fixes that are listed below. This feature list is not complete but gives a good overview of the changes and additions made in this release.

Licensing

Subscription based licenses

With the release of LS One 2014, LS Retail offers subscription licensing for all partners. Licenses can be paid on a monthly basis in a pay-as-you-go model with no commitment upfront.

To participate the partner shall have agreed to LS Retail’s terms and conditions, and a subscription addendum between the partner and LS Retail needs to be in place. A good credit history with LS Retail is also required and no pre-payment terms must have been agreed upon between the partner and LS Retail.

Perpetual licenses are also available.

Demo/Partner licenses

The new POS Demo license always expires three months after the LS One POS has been installed. The demo license is restricted and will not be able to perform the following operations: End of day, End of shift, Print X, Print Z, Tender declaration, Clear quantity, Void item, Return item, Return transaction, Customer search and it will not be possible to sell more than 6 items.

Partners can apply for partner licenses through the partner portal.

Transaction pushing from POS

In this release the LS One POS can trigger a scheduler job which replicates the all the transactional data from the terminal database to the store database.

Centralized returns

The operation Return transaction can now look up in all transactions stored at the Store/Head office. And when items are returned from a sale they are marked as such and cannot be returned again.

The POS first looks centrally. If the transaction does not exist there, it will look for it in the local database.

Hospitality improvements

Restaurant management

When guests are seated you can register how many guests have been seated at the table. This number can be displayed on the table in the table view.

A registered customer can be added to a table. The customer search is opened where a customer is selected and his/her name can be listed on the table in the table view.

For restaurants not using the table management system the Kitchen BOM can include an order number which also can be printed on the customer receipt, see images.

 


 

 

 

Unit of measure set on item sale

For items such as beer or wine which are sold in different size glasses or bottles but all have the same inventory unit, the POS can now request the POS user to enter which unit is being sold. So, when the item is sold, a dialog pops up where you can select a glass or a bottle of wine for an example.

 

Usability improvements

Stock counting

Stock counting has been made easier. You can either export a stock counting lists which can be filled in and then imported again or the stock counting can be done within the Site Manager where the Stock counting dialog has been modified to be filled in more easily by enabling barcode scanning and the entering of counted quantity.

Transaction search in Site Manager

The Search bar has been added to the Receipt browser in the Site Manager where you can search in all receipts with the following search parameters:

  • Date and time
  • Store
  • Employee
  • Receipt number
  • Terminal

Also the total amount has been added to the list of receipts to help with the search.

 

Authentication token login

Both in the LS One POS and Site Manager, the authentication token logins are now not scanned directly into the Login field. But first you have to select the card login button and then scan you authentication token. This increases the security as the token number can be hidden to the user.

The POS and Site Manager remembers the type of the last login and opens in the token login, if that was used to log in the last time.

Normal login screen at the POS – The card button has been added.

The new token login – there is a button to get back to the username and password login.

Note: Employee card setup has been removed and the authentication token should be used instead.

Numeric keypad

In the POS systems, numeric usernames and passwords are often used to speed up user login. On the small screens of some hardware it can be hard to enter the password on the touch-screen keyboard.

Now the user can switch to using numeric keypad to insert the username and password and the LS One POS remembers which was used for the last login. So if all users have numeric logins and passwords, the keyboard can always only have numbers.

 

Scale certification

LS One has been modified to fulfill the requirements of the WELMEC 2.2 guide for testing Point of Sale Devices connected with a Non-Automatic Weighing instrument. The certification process is in progress at the time of the release.

These changes involve displaying units such as kg and $ or € or all numbers shown on the POS screen, Customer displays and receipts. And some parts of the code that controls the connection to the scale, line display and printer have been locked with checksums. So if this code has been manipulated, a scale cannot be connected to the POS.

The POS About dialog includes all version numbers, checksums and will include the scale test certificate number when the test certification process is done. The POS will also display the certification number in its status bar when a scale is connected to it.

 

Z-report changes

It depends on where you are in the world what demands authorities do to a retailers Z-report. Retailers and restaurant owners also have various needs. Therefore the X/Z-report can be modified with some settings in the functionality profile.

  • The Sales and returns can be printed as a sum or divided into two and whether it includes the tax or not. This both applies to the Grand total sales and the sales for this report.
  • The X/Z-report can include the sum of returns
  • The X/Z-report can include the sum of all suspended transactions and how many they are.
  • X/Z-report can include a box of other information such as number of training transactions and number of receipt copies and invoices.
  • The tender report can include floats made during the day and tender declarations.

 

Tax free receipt printing

Now LS One POS can print tax free receipts on the hardware profile printer. The POS calculates the tax refund from some given ranges and does so for all tax codes on the sale.

There is no user interface in the Site Manager yet but the tax ranges can be set up with a database script.

Demo data
  • Demo reports have been fixed so that voided transaction are not calculated and displayed in the reports.
  • Replication data has been updated for the new tables and fields
  • New demo data package with American settings preset.
    Company currency is set to dollars, prices are shown without tax and calculations are done to prices without tax.

Customer

  • The number of training transaction can be printed in the x/z reports.
