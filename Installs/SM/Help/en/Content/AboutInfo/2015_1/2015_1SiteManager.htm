Site Manager Improvements

Administration

Administration Options > Sequrity - Active directory enabled setting for user creation in Site Manager is now disabled by default.

Currencies

An 'Edit company currency' button has been added to currencies view. When pressed it opens the Company card where company currency can be changed.

Customers

Customers view has been greatly improved. Attributes have been added to the search filter. More columns have been added to the view and indication colors for blocked customer. Some actions have also been added that allow users to easily add a customer to price group, tax group and discount groups. All actions, adding, editing and deleting customer can be performed by right clicking on customers in the list and several customers can be added to a group at once.

Dashboard

A Dashboard ticket that allows users to backup their local database has been created. From the ticket the user can configure the path to the backup storage folder and trigger backing up of the database. We recommend using an external drive or cloud location for the backup.

​

Excel import
  • Customer address fields was not imported correctly with excel import. The address entered was saved/overwritten as the default address on the customer. This has now been fixed.​
  • A simpler excel item import template has been created. This template has fewer fields that the old full template and is meant to be used with the LS One In-a-Box version. It will though also be available in the full version of LS One. A demo template with data will be provided with LS One Site Manager as well as an empty template. The user will be able to configure the path to the excel template folder.
  • A demo excel import template has been created for LS One In-a-Box. When the excel import working folder is configured this example file is created in the folder ..\_LS Retail\Default templates.
  • If a bar code setup and a bar code are imported through the excel import the import checks if the barcode matches the length of the bar code setup. We have made changes so that a shorter bar code should be accepted and the Excel import should therefor also allow shorter bar codes than the bar code setup requires.​
Forms & layouts
  • ​A new customer text field has been added to receipt designer (Header - Footer > Customer section). If there is no customer on the transaction lines are not displayed for this and all other customer fields.
  • All loyalty fields are now only visible if there is a loyalty card on the transaction. Lines are removed from transaction as well.
  • The Form info fields that can be edited on store and terminal (Labeled 'Form info 1 - 4') and can be added to receipt through receipt editor, now only create a line on the receipt if the field has some text.​
  • A new receipt type has been created for gift receipt. Gift receipt is a receipt that can be printed for sales transactions on demand. The receipt can omit prices and can be used to exchanges gift items. When the receipt is printed items that should appear on it can be picked from the original transaction. I.e. not all items need to appear on the gift receipt.
Inventory
  • Inventory status was not updated correctly when sales and inventory units were different. This has now been fixed. Now inventory is updated correctly even though sales, inventory and purchase units are all different. Both when an items is sold, counted, adjusted and purchased.
  • The system now does not allow changes to inventory unit of sold items without posting a statement first. This is so the inventory is up to date with the old unit before the inventory unit is changed.​
LS One Express

Error message is displayed if cloud authentication fails when logging into LS One Express for the first time.

LS One In-a-Box

Special Site Manager setup file has been created for the LS One In-a-Box version of LS One.​​

Permissions
  • Edit Company info is now connected to the 'Edit stores' permission.
  • Payment types button on Allowed payment types tab on Store card is now disabled if user has only 'View payment types' permission granted and 'Edit payment types' denied and removed if user has 'View payment types' denied as well the button is removed.
  • If 'Manage allowed payment types' is denied the user can access the allowed payment types to view them but has no write access and therefore can not change anything.
  • User with 'Manage reports' permission can now add and remove reports from Manage reports section in the Site Manager.
Reports

Report data pack has been updated with new reports. The reports included in the pack are described in the Report library section of the online help.

Terminal

Terminal groups were not used in the system and have therefore been removed.

Translations

Site Manager has been translated to the following languages:

  • German
  • Spanish
  • French
  • Belgian-Dutch
  • Swedish

Other improvements to translations:

  • ​Users view - some Icelandic translations changed.

  • Periodic discount priority view - Icelandic translations updated.

UX improvements
  • Profile and touch button selection fields are now automatically set in New store and New terminal dialogs, if there only exists one profile/layout in the database.

  • User interface refinements in various views and dialogs.

Other Bugfixes
  • When name of user is copied and edited the name changes in RBOSTAFFTABLE as well as in the USERS table.​
  • Fixed bug in user permission handling. Now users are not able to change permissions or passwords of user with higher permissions.
  • If you used a keyboard input barcode scanner to scan in a barcode when creating a new item in the new retail item dialog the scanner pressed the OK button and the user was unable to insert price or decide to press the add another button. ​This has now been fixed so that if a barcode is scanned the focus moves to the price field.
