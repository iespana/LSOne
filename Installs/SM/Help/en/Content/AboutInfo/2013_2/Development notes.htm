2013.2 Development notes

Customization additions

A new event has been added to the Events service; PreDisplayReceiptItem. If implemented the partner can control if an item is displayed in the receipt control.

The POS will still do a complete calculation of the transaction so, if the items not displayed have a price, additional customization will be needed in the POSCalculation service to not include those items in the total balance of the transaction.

Development Pack Improvements
  • Development Pack now works for Visual Studio 2012 and 2013.
  • Traning solutions and demo code have been updated and added to the development pack.
  • SiteManagerPlugin for the Site Service is now part of the development pack and can be built from there.
  • Install scripts have been updated so to that they can be compiled from the development pack with tese steps:
    • Bulid the development pack
    • Open the install scripts in InnoSetup from where they are locted in the development pack(LS One Dev pack 2013.2\Install Scripts\) and compile.

    This means that developers no longer need to move the install scripts to a new folder.
