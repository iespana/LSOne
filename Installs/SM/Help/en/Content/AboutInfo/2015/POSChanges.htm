POS Changes

  • A new operation was added to the POS called Item sales report. It will allow the user to select a date and then print a sales report on the receipt printer.
  • Run job operation now concludes the transaction properly so there is no need to log off after clicking the button. Also an indication of what is going on is displayed to the user.
  • Run job operation is also not allowed to be run while a transaction is in the POS.
  • Tender limitations can now be set on each allowed payment on the store card. If the POS calculates that the cash drawer exceeded the limit of the tender the cashier is notified.
  • Return transaction operation now works the same as the Return transaction button in the Journal i.e. only allows one transaction to be returned at one time.
  • Information in Price check dialog moved around.
  • New parameter added to operation Print Z report that determines if the operation End of day should be run automatically after the Print Z report is finished.
  • More than one suspended transactions can now be recalled into one transaction
  • When the user changes the password on the POS the Site service is now used to update the password change at the HO database.
  • A parameter has been added to the Set Qty operation which takes a number both positive and negative that will serve as the added qty. Look for +1 and -1 buttons in new layouts in demo data
  • The POS was allowing the user to change and clear qty on scale items as well as changing the UOM. This is not allowed anymore
  • If the POS shuts down (or the user clicks Log off force) while in training mode and a transaction is still active in the POS the next time the POS is started the POS will expect the user to handle the unconcluded training transaction (for example void it).
  • If the user tries to start the POS in training mode and there is an unconcluded transaction in the POS the POS will not allow training mode.
  • When returning an item and paying back into a card the POS will not display the cash-back dialog
  • At start up the POS will now clear everything out of PosisLog that is older than 30 days
  • The user can now right-click on a button and select „Add item“ which will automatically create a Item sale operation and the user only needs to add the item id/barcode
  • Messages displayed before logon dialog are now displayed center screen
  • Max return line and Max total return functionality reviewed and fixed
  • Line display showed the price of the item not the total amount to be paid for the item
  • User can enter a number with decimals in Tender removal dialog and Float entry
  • Return transaction allowed in sales trans even if Sales and returns are not allowed.
  • When paying with Credit memo the change back was printed twice on the receipt
  • Rounding enum on Allowed payment tab wasn‘t being read correctly by POS
  • Pictures on POS buttons sometimes went transparent
  • When the POS assigns a price to an item from the barcode it listens to the Keyed in price includes tax flag from the Store card
  • Blocked customer can now not use a loyalty card or pay with loyalty points
  • If business day file is empty the POS will create a new one at start up
  • Items that have partially the same barcode got mixed up on item buttons in POS layout
  • Error when printing an invoice from Show Journal
  • All log lines were saved to PosisLog with log level 1
  • Returning transactions that had M&M Least expensive, Discount offer or Discount amount were not being calculated correctly.
  • Cancel button on Income account operation in POS worked like an OK button
  • Token login functionality reviewed and some smaller issues fixed
  • Sum of discounts variables are now being printed if added to the receipt design
  • Decimal conversion for line display and printer has been changed. The change has been tested with Icelandic and Russian characters.
