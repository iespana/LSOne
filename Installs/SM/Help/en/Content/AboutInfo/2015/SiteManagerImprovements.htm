Site Manager Improvements

  • When in the retail items list the user can now update the variation combinations for all items in the filtered list by clicking „Create variation combinations“ under Actions on the right of the retail item list.
  • Replenishment excel import: When importing items that should not be blocked for purchase the blocked date does not have to be entered anymore.
  • Promotions were treated as a discount int he Customer card in Site Manager
  • Usage fields given a separate tab in Allowed payments settings.
  • Tender limitations for the cash drawer settings added to the Usage tab
  • Identification field on Customer increased to 20 chars
  • Price override trigger in functionality profile can be limited to only work for prices over a specific amount by setting the „Max selection“ setting on the Infocode to the minimum price that should be allowed before displaying the infocode
  • Gift cards can now be configured to be used as change back in a transaction
  • „New user“ has been changed; a server user is no longer available and an option to copy user settings from an existing user has been added. When copying user settings from existing user, permissions greater than the user creating the new user has are not added.
  • New statement dialog in EOD in Site Manager now selects by default to create a statement from last End of day.
  • Fuel item settings now have a separate tab in the retail items card
  • POS license plugin removed from Site Manager
  • Currency view now displays the company currency specifically and does not allow the user to change or delete it.
  • When creating or editing the company the Site Manager will detect if the company currency has an exchange rate as something other than 1 and will not allow the user to save the company
  • Language code on user POS settings was not being saved
  • Error when changing a Mix & Match error type on an existing M&M discount
  • When the last retail group was deleted from the database a null reference exception was thrown.
  • Use tax rounding in Price/tax calculations tab on Store card now has the default of false
  • The user could edit a promotion offer even if it had the status Enabled. This has been fixed
  • Token login functionality reviewed and some smaller issues fixed
  • A configuration (variant display setting) has been added to the variation group that decides how the variations should be displayed in the POS
  • Retail item search error fixed which only happened when the retail items list was ordered by something other than item id or description
  • Retail items list can now be sorted by price
