Update Scheduler Data for version 2017.1

When you receive a new version LS One you can either

  • delete the old scheduler data and import the new replication data package
  • or you can upgrade the old one.

To be able to upgrade you will need the following information

New tables

Create both normal and action jobs for this new table:

  • IMAGES

Also create a normal job for this table and set the Replication counter field to REPLICATIONCOUNTER:

  • RBOTRANSACTIONLOGTRANS

The default replication data package now contains

  • Normal job: N-IMAGES
  • Action job: A-IMAGES with replication action table REPLICATIONACTIONS
  • Action job: O-IMAGES with replication action table REPLICATIONACTIONS and will move actions. This is used to replicate to the Omni server.

Scheduler data update

To update your scheduler data from LS One version 2016.1 to version 2017 you will first have to do a Read design. Then you will have to create subjobs for the new tables and add them to the jobs being sent.

Run Site Manager version 2017.1 and log in to your database.

When you do this the first time your database will be upgraded to 2016 version. Make sure to take a backup before updating the database. New tables and fields are added to your database.

1. Do a Read design
  1. Go to Tools > Replication Settings > Distribution locations. Open the location from where you manage your replication and scheduling.
  2. In the right hand navigation press Read design

  3. Select Tables and fields option and press OK.
  4. Now the Data Director will run through your database design and update it to include all new fields and tables.
2. Add new subjobs

Add both action (A-) and normal (N-) jobs for all the new data tables listed above. On further information on how to do so read Subjobs

3. Add new subjobs to the jobs

Add the new action and normal jobs to the existing jobs. Read this topic: Jobs for more information on how to create and manage jobs.
