Development Notes 2016

We have upgraded LS One to use .NET Framework 4.6.1 which will require all partners to use Visual Studio 2015 for their customizations. This will also have the effect that LS One will only support Windows operating systems Windows 7 SP1 and later. Please see more information on supported operating systems on msdn.microsoft.com. Hardware requirements have changed due to this, please see the installation guides for more information on that.

InventTable, RboInventTable and InventTableModule have been merged into one table RetailItems. New data providers have been created for Retail items and the data providers using the older tables have been made obsolete.

InventDim, InventDimGroup and InventDimCombination have been discontinued and data providers using this table have been made obsolete

InventItemGroup has been discontinued and data providers using this table have been made obsolete

RboSpecialGroup and RboSpecialGroupItems have been discontinued and new tables SpecialGroup and SpecialGroupItems are being used instead. Data providers using the older tables have been made obsolete

All RboColors, RboStyles and RboSize tables have been discontinued. Data providers using these tables have been made obsolete.

Some other tables have also been discontinued. All data providers have been made obsolete in development pack so they cannot be used in development. These data providers will be removed in a later release but will remain in the development pack for this release.

Other development improvements
  • PostReturnItem trigger is now called right after the POS has been set to a "return state".
  • Dimension plugin is obsolete and has been removed from the Site Manager installation.
  • Partners can now create a service for address lookup.
  • Many dataproviders have been made obsolete due to the Variant framework addition. Most of the new dataproviders have similar names so they can be found easily. All the obsolete data providers now have a file name "Z-OBSOLETE<name of data provider> - OBSOLETE.cs" so it should be obvious which ones are not being used anymore.
  • The following assemblies have been removed from the Site Manager installer; HospitalityInterface, ForecourtInterface, ItemInterface, LoyaltyInterface.
  • The following plugins have also been removed; Configuration Wizard plugin, Infocode plugin.

  • DLLEntry property in most services has been made obsolete. This property should never be used unless it is absolutely necessary. All functions and dialogs should receive the entry (IConnectionManager) from the software that is using the service; Site Manager, POS, Site Service and etc.

Last updated: 27th April 2016
Version: LS One 2016
