Update Scheduler data to 2016.1

When you receive a new version LS One and need to upgrade your database and scheduler data you can either

  • delete the old scheduler data and import the new replication data package
  • or you can upgrade the old one.

To be able to upgrade you will need the following information

New tables

Create both normal and action jobs for these new tables:

  • IMPORTPROFILE
  • IMPORTPROFILELINES
Tables that should not be replicated anymore

Due to the inventory changes in this release the subjobs for the following tables can be removed as this information now flows through the LS Site Service

  • PURCHASEORDERLINE
  • PURCHASEORDERMISCCHARGES
  • PURCHASEORDERS
  • GOODSRECEIVING
  • GOODSRECEIVINGLINE

Scheduler data update

To update your scheduler data from LS One version 2016 to version 2016.1 you will first have to do a Read design. Then you will have to remove old subjobs from the jobs and at last create subjobs for the new tables and add them to the jobs being sent.

Run Site Manager version 2016.1 and log in to your database.

When you do this the first time your database will be upgraded to 2016.1 version. New tables and fields are added to your database. It is a good practice to backup your database before you upgrade it.

1. Do a Read design
  1. Go to General setup > Replication > Distribution location. Open the location from where you manage your replication and scheduling.
  2. In the right hand navigation press Read design

  3. Select Tables and fields option and press OK.
  4. Now the Data Director will run through your database design and update it to include all new fields and tables.
2. Add new subjobs

Add both action (A-) and normal (N-) jobs for all the new data tables listed above. On further information on how to do so read Subjobs

  • A-IMPORTPROFILE
  • A-IMPORTPROFILELINES
  • N-IMPORTPROFILE
  • N-IMPORTPROFILELINES
3. Delete unused jobs and subjobs

Delete the Inventory job and then delete all the subjobs for the data tables that do not need to be replicated anymore. These are:

  • A-PURCHASEORDERLINE
  • A-PURCHASEORDERMISCCHARGES
  • A-PURCHASEORDERS
  • A-GOODSRECEIVING
  • A-GOODSRECEIVINGLINE
4. Add new subjobs to the jobs

Add the new action and normal jobs to the existing jobs. Read this topic: Jobs for more information on how to create and manage jobs.
Below is a list of all the job changes that were made to the demo replication data package:

Preload store:

  • N-IMPORTPROFILE
  • N-IMPORTPROFILELINES

Store settings:

  • A-IMPORTPROFILE
  • A-IMPORTPROFILELINES

Missing subjobs

Subjobs for 4 new tables are missing from the replenishment data package and need to be added manually. Subjobs should be added to Item, Preload POS and Preload store jobs.

  • SPECIALGROUPS
  • SPECIALGROUPITEMS
  • DIMENSIONATTRIBUTE
  • DIMENSIONTEMPLATE
