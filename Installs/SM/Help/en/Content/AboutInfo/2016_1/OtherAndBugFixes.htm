Other improvements and bug fixes

Returnable flag

A new flag "Returnable" has been added to the retail item card. This property tells the POS whether an item can be returned or not.

The "Returnable" attribute has been added to the Attribute search filter in Retail items view.

Loyalty

"Local database" setting for loyalty in Administration Options has been removed. Loyalty is now always checked through Site service so if POS has no Site service profile loyalty functions will not work.

When returning a sale that has a loyalty point payment, information about the original payment is added to the returned sale

When editing a rule on a Loyalty scheme the Quantity/Amount field didn't show the existing value

If a loyalty scheme is used by one or more loyalty cards now it cannot be deleted

Demo data

New demo data has been created for version 2016.1

Bug fixes in LS One POS

Fixed bug in POS so now a customer can only be added once to the same loyalty card

Tool tip has been removed from barcode field in Price check dialog. The tool tip is meant to make amounts easier to read not bar codes.

If a retail group was set as a parameter to the Item search operation on POS an exception was thrown. This has been fixed.

Operation Manually trigger periodic discounts did not work if a periodic discount was selected as a parameter. Now the operation can trigger a specific discount when selected or display a list of discounts if no parameter is set.

Operation Item search now cannot take parameters to display prices as a default. The prices can still be displayed by clicking a button on the item search dialog.

The "max payment limitation" amount on a tender is now forced for the total amount of all payment lines in the POS of the same tender type

Sum line discount and Sum total discount variables in the receipt header or footer are now only printed when there are discounts on the sale. Line discount name and Total discount name variables are now translated.

Reprint receipt is now a POS permission and printing a receipt depends on that permission. If the user does not have this permission then the daily journal will display the receipt print preview but a copy can not be printed.

When items were being aggregated for printing the discount amounts were not being summed up when printing the receipt.

In the Manage database connection dialog, which is only visible when the POS is run in maintenance mode, the column marking a connection as the default connection is now the first column in the list.

When printing the receipt the currency symbol was not printed if the currency symbol was set as a suffix.

When an item is voided all linked items are now voided as well

Bug fixes in Site Manager

Site Manager now displays error message when job is run if source distribution location does not have a database design. Before the it seemed like the job was scheduled but nothing appeared in Data director job monitor.

Fixed database error when changing inventory unit on retail item.

Fixed an error that preventing the user from removing an item from a retail group through the Retail group card.

Purchase unit ID was not saved on variant items when they were created through the New retail item dialog. The fix included changing the data provider to return the inventory unit ID if the purchase unit ID is not available and if the item is saved the purchase unit ID is set to the inventory unit ID if it is still empty.

The list of columns that the user can choose from to design the receipt component on the POS user interface is now the same both in Site Manager designer and in the POS

Fixed a bug where both edit and delete buttons were enabled when opening the Styles view

Sum line discount and Sum total discount variables in the receipt header or footer are now only printed when there are discounts on the sale. Line discount name and Total discount name variables are now translated.

In the Scheduler view the enabled column in the database design view will now be visible on smaller resolutions

It is now only possible to enter 20 characters in the receipt display field in tax codes

In the Customer discount groups view and Item discount groups view a links to view and change the active discounts configurations has been added.
