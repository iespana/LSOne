Other improvements

In this section smaller improvements and bugfixes for both the Site Manager and LS One POS are listed.

Warning!!! The gift card receipt has to be updated when upgrading a customer to version 2017.2. The receipt was using incorrect variables which also let to coding errors. To get the updated receipt go to the Form profiles view and reset the default form profile.

 

If this is not done then when a gift card receipt is being printed on the POS an error will be thrown.

Name component

When entering a name into either the Site Manager or the LS One POS the name component has a selection box where a title can be selected i.e. Dr., Mr., Mrs and etc. These titles can now be translated. If the translation is left as an empty string then that title is removed from the selection box.

Site Manager improvements

Some older translations have been updated and new strings have all been translated into the languages that LS Retail supports out of the box

Inventory

Various smaller improvements were done in the Site Manager related to the inventory functionality. A description of all those changes can be found here.

 

Functionality profile

Start of day configuration has been simplified. Now there is only one check mark on the functionality profile to set that will tell the POS to use this functionality.

Added a new setting to the functionality profile that shows item prices by default when searching for items in the POS.

New infocode option has been added to the functionality profile where an infocode can be triggered on operation Open drawer in the POS.

Return infocode is now activated both for return item and return transaction operations but is not activated when selling an item with property "Quantity becomes negative"

A new option to select the "Main menu" for the Mobile inventory app has been functionality Profile, OMNI tab.

 

Miscellaneous

When creating a new retail item, if the user created a new inventory unit and selected it on the new item, it was possible to create an items without a sales unit.

Invoice customer can now be set when creating or editing customers on the POS.

Fixed an issues causing a crash when editing barcode mask segments

Fixed issues when modifying barcode mask segments in the Site Manager would not affect the underlying data.

When the user has the Site Manager in "Floating view" and closed the floating view by clicking the X in the upper right corner, the Site Manager didn't allow you to close the Site Manager application itself. This has now been fixed

A bug has been fixed where online help did not come up when pressing the help icon in the Edit configuration dialog.

Only tax groups with tax code are shown when choosing a tax group for customer, store, sales type and retail item. A tax group that is in use must contain at least one tax code.

In the Scheduler plugin all texts in the Job view (under the Log tab) have been changed to be normal casing

Images of type bmp can now be added to the image bank

Fixed an issue causing the application to crash when deleting a price discount group containing items.

An issue causing the transaction ID to be shown instead of the receipt ID in the receipt view header has been fixed

The credit limit field on the customer is now limited to 16 digits.

Fixed an issue where attributes for variant items were incorrectly displayed under header item 'Variant items' tab

POS layouts from versions 2017 and earlier can now be imported into the Site Manager. This issue was reported on the 2017.1 release.

 

LS One POS improvements

 

Some older translations have been updated and new strings have all been translated into the languages that LS Retail supports out of the box

Fixed an issue where the numeric text box had inverted input in the save drop, bank drop and tender declaration functionality when "use numpad" was selected in functionality profile

Return infocode (set in functionality profile) is now activated both for return item and return transaction operations.

Added a new setting to the functionality profile that shows item prices by default when searching for items in the POS.

An infocode can be configured to be trigged on operation "Open Drawer" on the POS. This can be configured on the infocodes tab on the functionality profile

Print Z operation is no longer allowed in training mode

Price override was not allowed for items that have a serial number. This has now been fixed

The inventory lookup dialog in the POS can now be filtered by region, the region that the current store is in is the default region that is displayed in the inventory lookup (called My region)

There was an incorrect message displayed when returning an item with mandatory serial number. This has now been fixed.

A bug has been fixed where selecting infocodes that had trigger function set to None threw an error.

Fixed an issue causing the POS to crash when modifying the layout that contains a logo panel and trying to save while the logo panel does not have any image.

Set quantity permission is now honored when numpad is used to enter quantity.

Fixed an issue allowing customer orders to be created without a customer, which resulted in a crash when recalling such customer orders.

Error that was thrown when buttons on the POS layout were changed from any operation to a payment operation has been fixed.

Fixed a bug where price calculations would return the wrong amount when tax exempting a prices that had been changed using the operation Price override.

Item line text and message panel information has been changed to accurately reflect the changes done when the price override operation is being run in the POS

In the Lock terminal dialog, if the user clicked the "Log off" button and that user did not have permission to run that operation the POS would still perform the operation. This has now been fixed

When items have been deleted and old receipts were being looked at in the POS included those items the descriptions of those items are now displayed in the POS. Previously "Item not found" text was being displayed.

Fixed an issues causing credit memos to not be issued correctly when change back on the payment tender was set to credit memo.

Fixed an issue where the return item infocode was popping up when selling an item with negative quantity.

Fixed a bug where price override amounts were not correctly calculated when a customer was set to the transaction with a different tax group than the store tax group.
