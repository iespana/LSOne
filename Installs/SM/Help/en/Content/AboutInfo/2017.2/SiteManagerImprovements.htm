Site Manager improvements

In this section general improvements that mostly or only affect the Site Manager are explained. Some changes may affect both the Site Manager and the LS One POS. For smaller improvements and bug fixes please see the other improvements section of the release notes.

 

Button menus

In order to be able to create the operation menus necessary for the Inventory apps some changes to how button menus were created in the Site Manager were needed. Now button menus can be found both under "Look and feel" and "Omni" sections in the "Sites" ribbon tab.

 

User interface changes

For the touch button layouts the button menu functionality remains the same as before but the user interface for the management of the menus and buttons has been changed. Now when entering the button menus a the first view is a list of all menu headers and a search filter to search for button menus. To view the buttons on each menu the edit button must be selected where information about the look and feel of the button menu is displayed and the tab "Menu buttons" has the list of buttons included in the menu.

 

The option of creating more than one button at a time (Add button in Menu buttons list) and the user being able to scroll between the buttons from within the button view, for an easier way of editing multiple buttons, has not been changed.

 

Functionality changes

When creating buttons for the POS touch layout the user should not see any changes but the main change is that operations that are available in LS One system are now marked if they can be used for All button menus, POS operation menus or OMNI operation menus. Which means that when creating button menus for Mobile inventory app only the operations that can be used in the app are available in the operations list. The same goes for when the POS menus are being created i.e. only POS operations are displayed in the operation lists (as before).

The "Main menu" option that can be selected on the button menus is a new configuration that is necessary for the Mobile inventory app. Only menus that have been marked as main menus can be selected as the menu that will be displayed in the Inventory app when it is first started. See changes in the functionality profile for further information.

 

Customer

The address tab on the customer card has been changed so that the address information can be viewed in an easier manner.

All addresses are now displayed in this one tab and they are marked as Shipping address or Billing address. The customer can have as many of each type as is necessary and now one shipping and one billing address can be marked as default. In previous releases the customer could only have one of each type of the addresses and the addresses that are already in the system will be set as defaults when the system is upgraded.

 

When a new customer is created and "Copy from" option is used the following fields are now copied from the selected customer to the new customer that is being created:

  • Invoice customer
  • Language code
  • Receipt option
  • Currency
  • Sales tax group
  • Blocking
  • Cash customer
  • Line discounts
  • Periodic discounts
  • Group limit
  • Customer groups
  • Infocodes
  • Sales prices
  • Promotions

 

The customer search in Site Manager can now be filtered by invoice customer.

Searching customers on the POS will also search for the invoice customer if there is any.

Fixed an issues causing the customer name to not be visible on multiple screen when the customer only had a display name set.

When adding a payment to a customer account through Site Manager a negative payment can now be entered.

Invoice customer can now be set when creating or editing customers on the POS.

 

User profile

In previous releases of LS One when setting up a user settings regarding maximum returns, maximum discount, store, visual profile, layout language settings and etc. had to be configured on each and every user. We have now created a user profile to make this setup easier for the store manager.

When the system updates itself these profiles will be created automatically from the information that is already on each of the existing users and the correct profiles attached to the users. An admin user profile will also be created along with the admin user when creating a new database.

When creating a new user a user profile has to be selected and the user profile is now visible on the user card (settings tab) instead of the configurations that were there before.

On the user profile card itself a list of users that are attached to the user profile can be seen and multiple users can be added to the use profile through that list as well as updating each user with a new profile.

 

Gift card balance receipt

A new receipt has been added to the system receipts called Gift card balance. This receipt is printed when the customer pays with a gift card. This receipt prints out information about the gift card with the amount that was paid as well as the remaining balance on the gift card after the payment if the gift card is a refillable gift card. If the gift card is not refillable then the remaining amount will always be 0.

To get the new receipt into existing data go to the Form profiles view and reset the default form profile.

If this receipt should not be printed then it can be configured to not be printed.
