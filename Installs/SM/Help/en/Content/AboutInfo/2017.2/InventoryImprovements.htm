Inventory improvements

Various changes and improvements needed to be done in order to accommodate the implementation of the Mobile inventory app. To see a description of the Inventory app functionality itself please click here.

 

Inventory templates

Previously inventory templates could only be created and used as a base for a worksheet for purchase orders. Now we have added an option to create a template and worksheets for stock counting. When the system is updated all previously existing templates and worksheets will be marked as being of type purchase order but moving forward the user must select what type of template is being created.

To accommodate the stock counting functionality and configurations the Inventory template view was changed both the functionality and user interface.

The properties that were previously under "Purchase worksheet settings" have now been divided up between the "General" tab and the "Purchase worksheet" tab. The general has configurations that will apply to any template created and then each type of template will have it's own tab with the specific configurations that apply to that type.

Item filter tab

The "Item filter" tab applies to all types of templates and the functionality remains as it was.

General tab

On the general tab a new configuration called "Unit selection" can be viewed where the user can select which unit will be selected by default when the worksheet is being created. The default value when a new template is created is "Inventory unit" which is the functionality as it was before this option was added. This configuration is now used in both purchase order worksheets and stock journals.

Other configurations were already existing on the inventory templates and their functionality has not changed.

Purchase worksheet

All configurations were already existing on the inventory templates and their functionality has not changed.

Stock counting

"Type of entering" is a new option where it can be controlled if a new line should be created in the stock counting worksheet when an item is added again to the same worksheet or if the quantity of the existing item should be updated.

The remaining configurations all have to do with the stock counting functionality on the Mobile inventory app.

Please note that the item filter functionality in the stock counting template is not applicable to the Mobile inventory app. But it is used when creating a new stock counting journal from a template.

"Quantity method" option controls if the Mobile inventory app asks the user for the quantity when doing stock counting or if the app sets a default value for the quantity (set in "Default quantity" option)

When doing stock counting in the Mobile inventory app the user can be asked to select an area where the stock counting is being done. This area can be any text and is not connected to any location or region in any way.

By clicking the "Copy from" button below the list, previously created areas can be viewed and used again. The area information is copied onto the new template so any changes done will not change the area values on the template where original area values are being used.

Stock counting

When creating a new stock counting journal a new option to create a journal from a stock counting template has been added. The user must select a template that has been previously created, of the type stock counting, and if the item filter has been set on the template the items will be added to the stock counting journal otherwise it is empty.

 

Regions

Regions can now be created and attached to stores. This option can be found on the Sites tab on the ribbon.

If the Site Manager is configured to be on a specific store then the "My region" option will display stores that are in the same region as the store the Site Manager is configured to be in. The same functionality applies to the Inventory lookup dialog on th ePOS.

If no regions are configured the functionality will remain as it was where all stores are displayed.

This functionality will be used more widely in future releases.

 

Permissions

New permissions have been added for viewing and managing inventory, purchase orders and goods receiving for all stores.

These permissions are called: "View inventory for all stores", "Manage purchase orders for all stores", "Manage goods receiving for all stores".

If these permissions are NOT granted and the Site Manager is configured to be on a specific store then the user can only create and manage these inventory documents for the store the Site Manager is configured to be on. If the Site Manager is configured to be at "head office" then these permissions are ignored.

Similar permissions for inventory adjustments, parked inventory and stock reservations already existed in the system.

Tips!!! The configuration to change the "current store" of the Site Manager is controlled by the permission "Maintain administrative settings"

Other improvements

Fixed an issues causing header items to be saved instead of variant items when editing a transfer order item line when receiving an order.

When changing the selection between Reservations, Parked and Adjustments (under Journal menu) the Site Manager screen "flashed". This has been fixed.

Created date filter in Reservations, Parked and Adjustments (under Journal menu) was not working properly. This has been fixed

Fixed an issues causing errors when selecting a variant item on the stock counting line dialog.

Fixed an issue causing the default vendor of an item to not be populated when selecting an item on the purchase worksheet dialog.

On hand inventory from the item inventory page in Site Manager and the inventory lookup dialog in the POS can now be filtered by region.

Fixed an issue causing purchase orders created from worksheets to have no description.

Fixed an issue where variant's description was not shown when editing a purchase order line that includes a variant item

Fixed an issue causing variants to be removed when editing the quantity on a transfer inventory request item.

When generating journals or stock counting, the staff ID that is currently logged in will be saved on the respective line. Related views can now be filtered by staff and area.

When a vendor, that has been previously deleted, is being imported through the Excel import an error message is now displayed in the Excel import report and the vendor is not imported.

Fixed an issue causing the default vendor to not be shown when previewing items on templates.

In the Excel import when items are being imported the casing of the unit ID's (PCS or pcs) is being ignored, and the correct unit id is now being found for the import regardless of casing. This issue was mostly causing issues in the stock counting functionality.

Search band "Created date" in all three journal views (adjustment, reservation and parked) had no effect on the search results regardless of what dates were selected. This has now been fixed
