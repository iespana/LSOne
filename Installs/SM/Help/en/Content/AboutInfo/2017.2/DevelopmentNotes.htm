Development changes and improvements

Development guide

More topics have been updated and added to the new development online help website.

Topics that we would like to specifically mention:

  • Comprehensive documentation on the Integration framework with samples.
  • Descriptions and detailed information of the services used by the POS.
  • Description of all components that the LS One team has created for the Site Manager and LS One POS
  • A chapter on how to do a Site Servicecustomization has also been added.

Information about updated and new topics is always displayed on the home page of the development help when new material is published.

 

.NET Framework

Note!!! LS One suite has been updated to run on .NET framework 4.7.

 

Upgrade issues

Warning!!! The gift card receipt has to be updated when upgrading a customer to version 2017.2. The receipt was using incorrect variables and changing the receipt to use the correct variables also meant that the printing code for this receipt had to be changed. To get the updated receipt go to the Form profiles view and reset the default form profile.

 

If this is NOT done then when a gift card receipt is being printed on the POS an error will be thrown.

 

 

Note!!! Due to changes done in the last release, LS One 2017.1, on how images are stored the exporting of POS layouts was also changed in that release to reflect the image bank changes. This means that exporting layouts from versions 2017 and earlier and then importing them into versions 2017.1 and later will result in the images not being imported or added to the buttons.

 

Customers

A new table CUSTOMER has been created and has replaced the table CUSTTABLE which has been depricated. When the system updates itself all the existing data in CUSTTABLE is moved to table CUSTOMER which only has the fields that the LS One system uses.

CUSTTABLE is a legacy table from when the LS One POS was the front end to the AX for Retail system so it had a lot of fields that the LS One system wasn't using.

Table CUSTOMERADDRESS has a changed primary key to allow for unlimited number of addresses for each customer. Previously this was limited to 3 addresses. The Primary key is now the ACCOUNTNUM and an identity field. Field AddressType was removed from the primary key.

 

User profile

New table POSUSERPROFILE has been created to hold the user profiles that can now be attached to users. When the system updates itself from an earlier release. The upgrade will create user profiles from the existing information on the users. If all the users have the same configurations then only one profile will be created and added to all the users. But for each variation of these configuration a new profile will be created.

 

Data Director Scheduler

Changed the scheduler to only look at the latest action for a given set of subjob and parameters. Previously it was over-processing the parameter values which could cause jobs to queue up on the scheduler while waiting for a large action-job to finish processing on the scheduler side.

 

LS One License information

The website where information about the LS One licenses for each customer has been updated. Now it displays information about both Site Manager and LS One POS licenses, when they were last validated, the description that was entered when it was activated, device name, Store ID and Terminal ID.

The website also displays if the license is of type Perpetual, Subscription or Partner license as well as information about how many licenses are active and how many are available.

To log onto the license website use the same credentials as are used when the LS One licenses are being activated on the Site Manager and the LS One POS.

To view the license information please go to the LS One License website

Tips!!! Licenses can be deactivated through this website by clicking the "Deactivate" button on each license line (only visible when the license has been activated)

 

Tips!!! To send requests for information or changes to the licenses to the LS Retail license department click the pencil icon displayed after the "License no" information and an email with the license information in the subject will be created.

 

 

Note!!! Only Site Manager licenses for versions LS One 2017 and later are visible on this website.

 

For information or changes to Site Manager licenses for older versions, you need to contact the license department at LS Retail directly.

 

POS operations table

A new table was created for the POS operations, OPERATIONS. This table will contain all the operations for both the POS and the mobile inventory app and therefore all the POS operations have been migrated to the new table.

Note!!! This table should never be used or changed in a customization and the POS takes care of populating and updating this table when a new database is created and/or updated. But if the previous table (POSISOPERATIONS) was included in a scheduler subjobs or demo data package those need to be updated.

 

Obsolete

Most properties that were marked as obsolete during the development of 2017.1 release have now been removed entirely. Any properties or functions that are currently marked as obsolete were marked as such during the development of the 2017.2 release.

 

Miscellaneous

The operation "Start of day" has been changed to be an internal operation and cannot be selected to be on a button on the POS layout. The operation is now run by the POS as an internal operation if the Start of day functionality has been activated in the functionality profile.

When performing the "Declare start amount" operation the POS now inserts a line into RBOTRANSACTIONPAYMENTTRANS that withdraws the entered amount from the default Float tender. This means that the operation now behaves like the "Float entry" operation.

A bug has been fixed where the background of the add/edit/delete context buttons in the Site Manager was white instead of transparent.

Verifone Point EFT integration for the USA is now included in the LS One release. Please see the Verifone US.Readme.pdf document on the partner portal on how to activate and configure the integration.
