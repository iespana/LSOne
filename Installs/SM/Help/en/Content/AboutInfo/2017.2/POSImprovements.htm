LS One POS improvements

In this section general improvements that mostly or only affect the LS One POS are explained. Some changes may affect both the Site Manager and the LS One POS. For smaller improvements and bug fixes please see the other improvements section of the release notes.

 

New POS operations

Clear discount

A new operation called Clear discount has been added. This operation clears out all manual discounts and manually triggered periodic discounts. Calculated discounts such as normal periodic discounts and customer discounts are not cleared. This operation has it's own permission with the same name.

Clear price override

A new operation called Clear price override has been added. A new permission was also added for this operation.

Gift card balance receipt

A new receipt has been added to the system receipts called Gift card balance. This receipt is printed when the customer pays with a gift card. This receipt prints out information about the gift card with the amount that was paid as well as the remaining balance on the gift card after the payment if the gift card is a refillable gift card. If the gift card is not refillable then the remaining amount will always be 0.

To get the new receipt into existing data go to the Form profiles view and reset the default form profile.

As with all receipts in the system the gift card balance receipt configured to not be printed.

 

Functionality profile infocodes

Infocodes that are configured on the functionality profile infocodes listed here below will now trigger linked infocodes when they configured.

Functionalities that will trigger a linked infocode (when applicable): Start/end of transaction, tender declaration, item not on file, item discount, total discount, price override, return item, return transaction, void item, void payment, void transaction and adding a salesperson.

Note!!! Linked infocodes can be used for Functionality infocodes, but infocodes with subcodes that triggers an item sale will not work in most instances. Items that are sold through infocodes need to be linked to an existing item and in many cases these infocodes have no association with an item. Please test your scenarios thoroughly when using infocodes with item sale subcodes.

Customer

Searching customers on the POS will also search for the invoice customer if there is any.

Invoice customer can now be set when creating or editing customers on the POS.
