Payment Type View

Site Manager > Store setup > Stores > Stores > Select store > Select Allowed payment types tab > Select payment type

What is the view used for?

  • View and add payment types to the store.

  • Each store should have at least one payment type set up and it is quite normal to have a few of them.

  • The settings for each payment type can be very different. Settings for payment types between stores can be different as well.
  • Remember that the payment types are set up separately for each store.
Fields & buttons Description
Store The ID number and name of the store is displayed here.
Payment method The payment type name.
Settings tab

General and Usage settings. Different fields that can be set up differently between tender types.

Fields & buttons Description
Payment method used for change The payment method used for change. This can be different to the original payment method.
Use another payment method Select the check box to allow the use of another payment method.
If amount higher than Select the amount that has to be reached for another payment method to be used. This is only available if the check box in the field above is selected.
Use payment method Select the other payment method to be used. Leave blank to allow any payment method. This is only available if the check box in the field above is selected.
Action

Select one of the options from the drop-down list:

  • Authorize card
  • Authorize card quick
  • Pay card
  • Pay cash
  • Pay check
  • Pay corporate card
  • Pay credit memo
  • Pay currency
  • Pay customer account
  • Pay gift card
  • Pay loyalty
Rounding

Values for the rounding field:

1 The number is rounded and displayed with no decimals.

0,1 The number is rounded and displayed with one decimal.

0,01 The number is rounded and displayed with two decimals.

Rounding method

Select one of the options from the drop-down list:

Rounding to next: Round up or down, whichever is nearer.

Always round up: Always round down to the nearest number.

Always round down: Always round up to the nearest number.
On payment Select the check box to open drawer on payment.
Blind count

Select the check box if the POS should not prompt the tender amount that should be in the drawer.

Max Tender declaration re-count

The maximum number of times that the user can re-enter the tender declaration if the POS produces an error message.

Max Tender declaration difference

The maximum difference between the tender declaration count and the tender amount that should be in the POS according to the transactions that have been recorded.

Usage tab
Fields & buttons Description
Counting required Select the check box so that this tender type is necessary for the tender declaration. This means that the tender declaration will not be allowed unless this tender type is counted.
Allow float Select the check box to allow float entry operation to be used on the POS.
Allow bank drop Select the check box to allow a bank drop operation to be used on the POS.
Allow safe drop Select the check box to allow a safe drop operation to be used on the POS.
Amount in POS limiting Check this field to activate the maximum amount check in the POS. If this field is checked the POS will notify the user when this payment type has reached the Maximum amount limit (see below).
Maximum amount Insert the maximum amount this payment type can reach in the drawer before the POS notifies the user. Then the user can remove some tender from the drawer by doing a safe drop, bank drop or removing tender.
Over tender / under tender tab
Fields & buttons Description
Can be part of split payment Select this check box if the payments can be split.
Over tender Select this check box if you want to allow over tender.
Maximum over tender allowed Enter the maximum over tender allowed.
Under tender Select this check box if you want to allow under tender.
Under tender amount Enter the maximum under tender allowed.
Payment limitations tab

Field that controls maximum and minimum payments, if payment type can be voided and negative.

Fields & buttons Description
Maximum payment allowed Select the maximum payment allowed.
Minimum payment allowed

Select the minimum payment allowed.

If the amount buttons on the payment dialogs should be limited to f.ex. 1 then 1 has to be entered here as a minimum payment for the payment type open.

Payment type can be voided Select this check box allow the payment type to be voided.
Allow negative payment amounts Select this check box to allow negative payment amounts.
Limitation code Displays list of available limitation codes to select from.
Allowed card types tab

For use in connection with card payments. To be able to add allowed card types to the payment method the card type has to be created in General setup > Payments > Card types

Fields & buttons Description
Card type The card type.
Modulus check

If this column is marked with a tick, a modulus check is done on the POS upon payment.

Check expiry date

If this column is marked with a tick, the expiry date is checked on the POS upon payment.

Internal authorization

If this column is marked with a tick, internal authorization within the LS One system is required upon payment. This is used by the loyalty cards, customer cards or any other card that is registered or created only in the LS One system.
Enter manually

If this column is marked with a tick, details must be entered manually upon payment.
Card types... Select this button to see and edit the list of card types available and to add new ones. Opens Card types view.
Edit Highlight the line and select the edit button to edit the allowed card type. Opens Allowed card type dialog.
Add Select this button to add a new allowed card type. Opens Allowed card type dialog.
Delete Highlight the line and select the delete button to delete the allowed card type.
