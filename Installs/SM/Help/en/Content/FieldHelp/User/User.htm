User View

Users > View all users > Select user

What is the view used for?

  • Edit the settings for individual users.
Fields & buttons Description
Login name The login name given to the user when created.
Name The name of the user, you can edit this by clicking on the text boxes.
Is domain user

If the check box is selected, this user is a domain user, if it is clear, they are not a domain user. Users have to be created as a domain user.

User is disabled If the check box is selected, this user has been disabled, and therefore cannot be used. If it is clear, the user's login is available.Users can be disabled because they have entered their password incorrectly too many times.
Link: Change password

Click on this link if the user's password needs to be changed. See more information on Change User Password.
Link: Authentication tokens Set tokens that can be used by the user to log in to the system. Opens Authentication tokens dialog.
Link: Disable Login Disable this user from logging in. If the user enters the wrong password more than the allowed attempts (see security tab in Administration Options), his login becomes disabled. On this link then it will be displayed Enable, so the administrator can enable his login again.
Link: User profiles Clicking on this link opens UserProfilesView.
Link: Receipts Clicking on this link opens Find receipt view.
Link: Group Permissions Clicking on this link takes you to the Group Permissions dialog.
Link: View all users Clicking on this link takes you to the list of users.
User groups tab

This tab shows a list of user groups available. A user inherits permissions from the user groups selected. These can be individually overridden in the Permissions tab.

Fields & buttons Description
Administrators

Select the check box next to the user group to place the user in the group.

Note: The User box relates to the user themselves, not the tabs, so if you press delete it will delete the user.
Other groups Create more groups in User groups. Go to Security > User groups > New user groups.
Permissions tab

Permissions are inherited through user groups. They can also be individually adjusted, so user group permissions are overridden.

Fields & buttons Description
Search Enter text into this box and click on the search button to search for a permission.
Clear search Click on this button to clear the search field.
Permissions The permissions tab allows you to search and edit the permissions granted to the user.
Granted from groups The permission has been inherited and granted from the user group.
Granted The permission is granted.
Denied from groups The permission has been inherited and denied from the user group.
Denied The permission is denied to this user.
Permissions categories

The list is organized into permission categories which should help you find the relevant permissions:

  • Data Director Monitoring
  • General
  • Hospitality
  • Inventory
  • Item Master
  • POS permissions
  • Profiles
  • System Administration
  • User and security management
Button:
Inherit 		Click on a permissions line that is not Denied and click on the Inherit button to give the user the permission highlighted. Note that this will only work if the permission is granted to the user group that the user belongs to.
Button:
Grant		 Click on a permissions line and click on the Grant button to give the user the permission highlighted.
Button:
Deny		 Click on a permissions line and click on the Deny button to deny the user the permission highlighted.
Settings tab

View and edit the settings for the user.

Fields & buttons Description
Name on receipt Enter or edit the name that appears on receipts when the POS user is logged in. This can be different to their description.
Email Enter or edit the email address of the user.
User profile Choose the user profile that you want to assign to the user from the the drop-down menu.
Edit Edit the selected user profile.
