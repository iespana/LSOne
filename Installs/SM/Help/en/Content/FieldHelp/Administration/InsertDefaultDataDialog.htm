Insert Default Data Dialog

Tools > Administration > Options > Administrative Options Box > Insert default data

What is the dialog/view used for?

  • Use this dialog to insert default data packages or the entire demonstration data.
  • Demo data is a setup that is based on a fictitious company, through which you can see examples for most of the operations on the Site Manager and POS.
  • We recommend using the default data packages to those who have never used the LS One suite before, and for those who want a fast set up. For an example there is a package for the POS Touch button layouts.

Quick tips:

  • To enter all default data, only select the Demo data check box.
  • Editing or erasing default data, after it as been entered, has to be done manually.
  • The default data is based on a fictitious store and may not have he correct setting for your business.

 

Tab/Field name Description
Omni data This data package includes all settings needed to connect Omni devices with either the LS Omni Mobile Inventory app or LS Omni Mobile POS. It includes a terminal and profile setup, a menu setup for the Mobile Inventory app and other settings needed. Use this package as a foundation to build your mobile inventory menus on and help you with the mobile POS setup.
Express template data This data package is intended as a starting data package for setting up a new in an hosted environment. It includes all standard settings that are needed to start a new store.
System report data In the Site Manager Retail > End of day there are operations button Financial report and Financial report by statement , these reports have to be imported with this package.
Hospitality data

Includes the button setup for the hospitality specific menus such as Split bill and Transfer table.

Known issue: You get a notification that not all data was inserted but all the buttons are inserted.

Touch button layouts Insert the default data for touch button layouts. This gives you four examples of touch button layouts. The look and feel of the layouts was updated in LS One version 2015
Currency data Insert the default data for currency. Note that not all of the currency data is available to you if you have not yet selected a Default tax calculation store in the Store Management tab of the Administrative settings in the Site Manager.
Profiles Insert the default data for profiles. This gives you demonstration profiles for Visual profiles, Functionality profiles, Hardware profiles, and Site Service profiles.
Bar code data Insert the default data for bar codes.
Demo data

The Demonstration Data is based on a fictitious company, Aurora International Ltd. This company is located in France and its company currency is set to the Euro. All the data in this company is fictitious and not necessarily correct in all countries. The sales-tax setup in this company is fictitious and does not imitate a real country’s setup.

The demonstration company data includes:

  • Four stores, of which one is a restaurant;
  • Two retail department;
  • Many retail groups;
  • Over 150 items.

This data package can be used for training, testing and demo purposes.

Replication data Insert the default data for replication. This inserts the scheduler data that you can find in the General Setup of the Site Manager. Note that you cannot insert replication data if you do not have the data director installed.
