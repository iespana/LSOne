Multibuy View

Site Manager > Retail > Price/discount > Multibuy

What is the view used for?

  • View and edit multibuy offers.
  • Activate and deactivate multibuy offers.
Multibuy offers - upper pane

In the upper list all multibuy offers are listed.

Fields & buttons Description
Offer number Displays the ID of the multibuy offer.
Description Displays the multibuy offer’s description.
Status Displays the multibuy offer’s status: Enabled means that the offer is active and in use, Disabled means that the offer is inactive and available for editing. (See Button: Disable/Enable for more information).
Discount type

Displays the discount type:

  • Unit price:
    The retail price for one item. For an example if you buy at least 5 pieces then you get each for 2 euros in stead of 2,5.
  • Discount %:
    The multibuy discount from the retail price. For an example if you buy at least 5 pieces then you get 10% discount of them all.
Priority

Displays the priority of the offer.

The priority of the offer controls which offer is triggered if the item exists in more than one offer of the same type. The lower the priority number, the higher the priority of the offer. For example, if the priority number is 1, it will have a higher priority than an offer that has the priority number 2. An overview of all discount priorities can be seen in periodic discount view
Validation period Displays the Validation period. Validation periods are set up in validation period dialog.
Starting date Displays the starting date of the validation period.
Ending date Displays the ending date of the validation period.
Price group Displays the price group set to the multibuy offer. The multibuy offer is active in the stores in the selected price group. If the field is empty the offer is valid in all stores.
Customer connection Displays a customer group or a single customer the offer is valid for. If this field is empty the offer is valid for all customers.
Triggering

Displays if the offer is:

  • Automatic: that is the discount is given automatically as soon as all the items needed to trigger this discount have been sold.
  • Manual: to trigger the discount the user needs to press a button on the POS. The operation of the button should be set to Manually trigger periodic discount. When this button is pressed all periodic discounts that are set to manual and apply to this transaction are activated.
Button:
Disable/Enable

Highlight a line and press this button to deactivate or activate a multibuy offer. If the line displays the offer’s status as Enabled, the button will be set to deactivate. If the offer’s status is Disabled the button will be set to activate.

Note that you can only edit an offer if the line is disabled and it will only work on the POS if the line is enabled.
Edit Highlight a line and press this button to edit the configuration of the multibuy offer in the multibuy dialog. This includes setting its priority, discount, price group and validation period.
Add Select this button to create a new multibuy offer.
Delete Highlight a line and press this button to delete the selected multibuy offer.
Multibuy lines - lower pane

In the lower list the lines of a selected Multibuy offer are displayed.

Fields & buttons Description
Type Each line is of a specific type. Item and variants can be added to a multibuy offer.
Item relation This is the ID number of the type, if it is an item, it will be the item ID, if it is a Retail group, the retail group’s ID is displayed, and so on.
Description Displays the description of the line type (Item , Retail group, Retail department, All, or Special group).
Variant This field is only visible if any of the lines include variant items. Then this field displays the variant description of the item.
Edit Highlight a line and select this button to edit the line by opening the Multibuy line dialog.
Add Select this button to create a new line by opening the Multibuy line dialog.
Delete Highlight a line and select this button to delete the selected line.
