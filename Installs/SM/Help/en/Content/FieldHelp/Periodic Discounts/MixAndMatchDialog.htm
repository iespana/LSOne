Mix and Match Dialog

Site Manager > Retail > Price/discount > Mix and match > Select Add button

What is the view used for?

  • To create a new mix and match offer. 
  • To edit an existing mix and match offer.
Fields & buttons Description
General  
Description Enter Mix and Match promotion description.
Priority The priority of the offer controls which offer is triggered if the item exists in more than one offer of the same type. The lower the priority number, the higher the priority of the offer. For example, if the priority number is 1, it will have a higher priority than an offer that has the priority number 2. An overview of all discount priorities can be seen in periodic discount view.
Price group Select a price group from the options available on the list. The offer will only be active in the stores in the selected price group. Edit the price groups in: Site Manager > Retail > Price/discount > Price/discount groups. If no price group is selected the offer will be active in all stores.
Add Select this button to add a new price group.
Account code Select an account code:
  • None - The offer is valid for all customers.
  • Customer - The offer is valid for the customer selected in next field.
  • Group - The offer is valid for all customers in the customer group selected in next field.
Account selection Select a customer or customer group the offer should be valid for (depending on the Account code).

Discount %

Line specific

Select one of these price/discounts. The fields on the right-hand side are active depending on the line that you select. You can select one of five options:

  • Deal price: The items required in the mix and match offer are sold for a fixed price that is defined in the Deal price field.
  • Discount %: When an item that triggers the mix and match offer is selected, this percentage is discounted from its total value.
  • Discount amount: When the mix and match offer is triggered, the value entered here is discounted from the item’s total price.
  • Least expensive: The discount, which equals the price of the least expensive item, is then proportionally distributed to all of the lines. Use this options for offers like buy 3 for 2 or buy 1 get 1 free.
  • Line specific: The price and discount are set up with the offer lines.
Triggering

Select if the discount should be:

  • Automatic: that is the discount is given automatically as soon as all the items needed to trigger this discount have been sold.
  • Manual: to trigger the discount the user needs to press a button on the POS or a barcode scanned. The operation of the button should be set to Manually trigger periodic discount. When this button is pressed all periodic discounts that are set to manual and apply to this transaction are activated.
Barcode If triggering is set to Manual a barcode can be scanned to trigger the offer. Enter the discount barcode here. Read more about additional setup for this here.
Price / discount  

Radio button selection:

  • Deal price
  • Discount %
  • Discount amount
  • Least expensive
  • Line specific

Mix and match offers can be of various types.

  • Deal price - all products needed to trigger the offer will be sold at a fixed price
  • Discount % - all products needed to trigger the offer will be sold at discounted price
  • Discount amount - all products needed to trigger the offer will share a fixed amount discount.
  • Least expensive - the x number of least expensive products are given as a discount amount and the discount is distributed between all products needed to trigger the offer.
  • Line specific - each item needed to trigger the offer can have various discount setup.
Deal price If Deal price was selected in the previous field enter the total price for all the items in the offer.

Discount % value

 If Discount % was selected in the previous field enter the discount percent value that will applied to all items in the offer.
Discount amount value If Discount amount was selected in the previous field enter the amount to be subtracted from the total amount of all items in the offer.
Number of least expensive lines If Least expensive was selected in the previous field enter the number of least expensive items that should be for free.
Validation period  
Validation period Select a validation period from the list.
Edit Select this button to edit the selected validation period in the validation period dialog.
Add Select this button to add a new validation period in the validation period dialog.
Starting date Displays the starting dates for the validation period. To edit, select the Validation period edit button.
Ending date Displays the end dates for the validation period. To edit, select the Validation period edit button
