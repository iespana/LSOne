Multibuy Setup Dialog

Site Manager > Retail > Price/discount > Multibuy > Select the Add button/Highlight a multibuy offer and select the Edit button

What is the dialog used for?

  • To create a new multibuy offer.
  • To edit an existing multibuy offer.
Fields & buttons Description
General  
Description Enter the description of the multibuy offer.
Priority The priority of the offer controls which offer is triggered if the item exists in more than one offer of the same type. The lower the priority number, the higher the priority of the offer. For example, if the priority number is 1, it will have a higher priority than an offer that has the priority number 2. An overview of all discount priorities can be seen in periodic discount view.
Discount type

Select the discount type:

  • Unit price:
    The retail price for one item. For an example if you buy at least 5 pieces then you get each for 2 euros in stead of 2,5.
  • Discount %:
    The multibuy discount from the retail price. For an example if you buy at least 5 pieces then you get 10% discount of them all.
Price group Select a price group from the options available on the list. The offer will only be active in the stores in the selected price group. Edit the price groups in: Site Manager > Retail > Price/discount > Price/discount groups. If no price group is selected the offer will be active in all stores.
Add Select this button to add a new price group to the list.
Account code

Select an account code:

  • None: The offer is valid for all customers.
  • Customer: The offer is valid for the customer selected in next field.
  • Group: The offer is valid for all customers in the customer group selected in next field.
Account selection Select a customer or customer group the offer should be valid for (depending on the Account code).
Triggering

Select if the discount should be:

  • Automatic: that is the discount is given automatically as soon as all the items needed to trigger this discount have been sold.
  • Manual: to trigger the discount the user needs to press a button on the POS or a barcode scanned. The operation of the button should be set to Manually trigger periodic discount. When this button is pressed all periodic discounts that are set to manual and apply to this transaction are activated.
Barcode If triggering is set to Manual a barcode can be scanned to trigger the offer. Enter the discount barcode here. Read more about additional setup for this here.
Validation period  
Validation period Select the validation period from the list.
Edit Select to edit the selected validation period in the validation period dialog.
Add Select the add button to add a new validation period.
Starting date Displays the starting dates for the validation period. To edit, select the Validation period edit button.
Ending date Displays the end dates for the validation period. To edit, select the Validation period edit button.
Configuration  
Minimum quantity

Displays the minimum quantity of multibuy lines that need to be sold to trigger the multibuy offer.

Note: that this is the minimum quantity, if this is exeeded, the multibuy offer will continue discounting in the same way.
Unit price/Discount %

The name of this column depends on the selection in the discount type field.

  • Unit price: Displays the unit price for the item or variant.
  • Discount %: Displays the discount percent that is applied to the multibuy lines when the minimum quantity is sold on the POS.
Add Select this button to add a new multibuy configuration in the multibuy configuration dialog. You can add many configurations to the same offer.
Delete Highlight a line and select this button to delete the multibuy configuration.
