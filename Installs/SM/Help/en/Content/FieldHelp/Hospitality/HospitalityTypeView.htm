Hospitality Type View

General setup > Hospitality > Hospitality types > Highlight line and select Edit button

What is the view used for?

  • View and edit settings for the hospitality type.
Fields & buttons Description
Restaurant ID Displays the restaurant ID.
Sales type Displays the sales type. Edit it by selecting another from the drop-down-list menu. To create a new sales type, go into Retail > Retail items > Sales types > Select Add button
Description Enter or edit the description of the hospitality type.
View all hospitality types Click to open hospitality types view.
General tab
Fields & buttons Description
Staff takeover in transaction
  • Always - If another staff member opens a table the table will always be registered to this staff member
  • With confirmation - The staff member will be asked if he/se wants to take of the table when editing the table.
  • Never - Another staff member can edit the table with out being registered for it.
Touch buttons Each Hospitality type can have its own Touch button layout. If it is not set, the POS selects the layout depending on Store, Terminal and Staff layout configuration.
Edit Select this button to edit Touch buttons, opening Touch button layouts view.
Top POS menu ID The look of the button the user clicks to select the current Hospitality type.
Edit Select this button to edit the POS menu ID, opening the Hospitality POS menus view.
Add Select this button to add a POS menu ID, opening the New POS menu dialog.
POS Logon Menu ID The button menu on the right side of the Table view on the POS.
Edit Select this button to edit the POS Logon Menu ID, opening the Hospitality POS menus view.
Add Select this button to add a POS menu, opening the New POS menu dialog.
Stay in POS after transaction

If the check box is clear, the POS will automatically return to the Table view when the transaction is finished.

If it is selected, the user will always have to select the Exit Hospitality POS operation in the POS to return to the Table view.
Default type Select this check box to make this type the default landing view on the LS One POS.
Send voided items to station If it is selected, the voided items will be sent to the restaurant stations.
Send transfers to stations If it is selected, the items transferred to another table will be sent to the restaurant stations.
Send suspensions to station If it is selected, the suspended transaction will be sent to the restaurant stations.
Send order number to station If it is selected, when items are sent to the restaurant stations, the order number is sent as well.
Send to station

Controls when the POS asks the user whether items should be sent to the Restaurant stations.

  • None: The POS will not send any thing to printing stations.
  • At POS exit: Station printing happens when “Exit Hospitality POS” operation is run. That is when going back to table view.
  • At POS posting: Station printing happens when the transaction payment is concluded.
  • At POS exit or POS posting: Station printing is done both when the transaction payment is concluded and when the “Exit Hospitality POS” operation is run.
  • At item added - Not implemented in current release.
  • At item added one delay - Not implemented in current release.
  • Always print all - All items are sent to printing stations at POS exit or at POS posting without asking.
Table button tab
Fields & buttons Description
Update table from POS Select the check box to make the POS update the table centrally without going back to the table view. This is done with the time interval set in the field Table update timer interval which is in the Hospitality setup view.
Dining table layout id Select a table layout ID from the drop-down-list menu
Edit Select this button to edit the dining table layout selected, opening the Dining table layout view.
Add Select this button to add a dining table layout, opening New dining table layout dialog.
Table button description

The display on the table buttons:

  • Number: Displays the table number only.
  • Description: Displays the table description only.
  • Number + description: Displays the table number followed by the table description.
  • Description + number: Displays the table description followed by the table number.
Table button staff description

Choose how to display the member of staff that last used the table:

  • Staff id: Display the staff ID number on the table buttons.
  • Receipt name: Display the receipt name on the table buttons. This can be edited or viewed in Site Manager > Security > Users > View all users > Double click on user > Select POS settings tab
Add customer to table If selected, the POS will display the Customer search dialog (when clicking on "Seat guests", on "Run POS" or double clicks on an available table) where the user can search and select a customer to be added to the table.
Customer name on table

Choose how the customer name should be displayed on the table button:

  • No customer information displayed: Only the table button description will be shown.
  • Only customer name: Displays the customer name only, if there is a customer on the table.
  • Customer name + table button description: Displays both customer name and the table button description.
Request number of guests If selected, the POS will display a numpad dialog where the user can insert the number of guests to be seated. The number of guests will be visible in the lower right corner of the table button.
Max number of guests Enter the max number of guests allowed to be seated per table. The default is set to 12.
Split bill/Transfer lines tab

Select menus for split bills and transfer lines.

Tip: To take a closer look at menus, go back to the hospitality view and select “Hospitality POS menus” where you can preview all of the individual menu parts.

Fields & buttons Description
Split bill POS menu group Select a Split bill POS menu group from the drop-down list.
Edit Select this button to open Menu group dialog and edit the menu group selected.
Add Select this button to open Menu group dialog and add a new menu group.
Transfer lines POS menu group Select a Transfer line POS menu group from the drop-down list.
Edit Select this button to open Menu group dialog and edit the menu group selected.
Add Select this button to open Menu group dialog and add a new menu group.
Max no. of splits Enter the maximum number of splits that a transaction can be split by.
Combine split lines action

If a the bill for an item is split, and then some parts or all are put onto the same bill, choose whether the item parts are combined, or if they stay separate.

  • Never combine: Do not combine item parts.
  • Always combine: Combine item parts.
  • Combine on confirmation: Combine if the user confirms that they want to.
