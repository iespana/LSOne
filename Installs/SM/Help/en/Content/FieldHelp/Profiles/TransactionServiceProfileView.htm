Site Service Profile View

Store setup > Profiles > Site service profiles > Edit a site service profile

What is the view used for?

  • To setup a connection to a Site Service and to send a new configuration to a Site Service.
Fields & buttons Description
ID Displays the ID number of the site service.
Description Enter or edit the description of the site service in this text box.
Connection tab
Fields & buttons Description
Site service host Enter the name or IP-address of the host where the Site service is running.
Site service port Enter the Port number the Site service is running on on the host machine.
Button:
Test connection		 Select this button to test the connection to the site service.
Button:
Stop site service		 Select this button to stop the site service. The site service has to be stopped and restarted to load new configuration settings.
Button:
Start site service		 Select this button to start the site service. The site service has to be stopped and restarted to load new configuration settings.
Button:
Configuration ...		 Select this button to edit the configuration settings for the site service in the Site Service configuration dialog. If the configuration is changed the new configuration has to be sent to the site service and the service has to be stopped and started again.
Gift card tab
Fields & buttons Description
Use gift cards Select this check box to allow the use of gift cards.
Gift card option

If the gift card check box is selected, select one of the options from the list to define how the gift cards can be used:

  • ID - gift cards are issued with a fixed amount in the Site Manager before they are sold in the LS One POS. This setting is designed for preprinted gift cards which are registered in the Site Manager before they are sold on the POS.
  • Amount - LS One POS issues the gift card at the same time that it is sold. The gift cards gets the next available ID from the number sequence. This setting is designed for gift cards printed at the time of sale.
  • ID and Amount - This option issues the gift card at the same time as it is sold and the user enters both the gift card amount and ID number. This setting can be used with preprinted or plastic gift cards which are registered and filled with the amount at the time of sales.
Refillable

Default yes - If a gift card is created at the POS or at the Site Manager, the new gift card will be refillable.

Default no - If a gift card is created at the POS or at the Site Manager, the new gift card will not be refillable.

Maximum amount The maximum amount allowed for a single new gift card.

Customer tab
Fields & buttons Description
Use centralized customers Select this check box to use centralized customers. This is a customer that can be created on the POS, the information is saved centrally at Head Office and in the local database so that the user can add the new customer directly to a transaction but the other POSs need to wait for the data replication to get the information about the new customer.
Default sales tax group Choose the default sales tax group for the customer from the list.
Edit Select the edit button to edit the default sales tax groups available on the list. Opens Customer and store sales tax group view.
New customer is cash customer Select one of the following options from the list: Always yes / Always no (the value is set on the Site Manager and the POS user cannot change it) Set on POS default yes / Set on POS default no (The value is preset on the Site Manager but the user can change it on the POS)
Allow manual input of ID Select this check box to allow manual input of customer ID when creating new customers in the POS.
Default credit limit Set the default credit limit that will be set to new customers created in the POS.
Mandatory fields Check which fields are mandatory to set when creating or editing customers in the POS.
Central returns tab
Fields & buttons Description
Use central returns If this box is checked then the POS will look first for return transactions centrally and then locally if not found centrally.
Email receipts tab
Fields & buttons Description
Email receipts Select this checkbox to enable the emailing of the customer receipt and other printable forms. Read further about email setup here.
Options

Select one of the following options:

  • Never - emails are never sent
  • Always - any transaction that prints a form will be sent. If a customer with an email has been added to the transaction this email is used. But if an email address is missing then the POS requests for an email address.
  • Only to customers - receipts will only be send to registered customers with email address listed.
  • On request - An email receipt will only be sent if it is requested after the transaction. An email receipt can be sent for any finalized transaction by finding it by receipt number.

See more information on this here
Email shorthand options An email shorthand is a button that will be included in the email dialog in the POS. You can create buttons with common email address endings to speed up the email address capturing. An example of email shorthand is for an example: @gmail.com, .com, @.
Edit Select an existing shorthand from the list and press this button to edit it.
Add Press this button to create a new shorthand button
Delete Press this button to delete the selected shorthand from the list.
