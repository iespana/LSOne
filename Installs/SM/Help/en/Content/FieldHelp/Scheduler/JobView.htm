Job View

General setup > Replication > View Jobs > Edit a job

What is the view used for?

  • To view or edit jobs, to add subjobs to the job and set triggers.
  • A job is a unit of replication. It has source locations that specify where to replicate from and a destination location that specifies where to replicate to. Jobs have list of subjobs. Each subjob represents a database table that should be replicated or a stored procedure that is executed to replicate data. To perform replication through a job, the job must be run. Jobs can be run manually or scheduled to run at a specific time or at specific time intervals.

 

Fields & buttons Description
Description A short description of the job.
Job type The type of the job. The only possible value for this field is Data replication. Future versions might hold more job types.
Source location The source distribution location for the job. All data that is replicated in this job will be read from the database in this location. The source location can be a group location; in this case data will be read from all locations in the group that have database information defined.
Destination location The destination distribution location for the job. All data that is replicated in this job will be read from the source location database and inserted into this database. The destination location can be a group location; in this case data will be sent to locations in the group that have database information defined.
Enabled

Determines if this job is enabled and can be run. Uncheck this to hide this job from the job list and prevent it from being run.
Use current location

If checked then the value in the Current location field, found under Site Manager Options Configuration, is used as source location.
External command If checked a tab called Parameters is opened. If applicable then a name of a plugin and a list of procedures included in that plugin is displayed.
Subjobs tab

A list of all subjobs included in the job.

Fields & buttons Description
Use subjobs from job You can run sub jobs included in another job than the job selected by selecting the job in this field. When another job is selected the subjob list displays the subjobs included in this job and the list is grayed to
Description Displays the description of the subjob
Method

Displays the method of the subjob. There are 3 types of subjobs:

  • Normal
  • Action
  • Stored Procedure
Table/Procedure The name of the database table or stored procedure the subjob replicates.
Database design Displays the database design the database table belongs to.
Move up Select this button to move the selected subjob up in the list. The subjobs are replicated in the order they appear in the list.
Move down Select this button to move the selected subjob down in the list. The subjobs are replicated in the order they appear in the list.
Edit Select this button to edit the selected subjob in the subjob view.
Add Select this button to add subjobs to the job in the add subjobs dialog.
Delete Select this button to delete the selected subjob from the job.
Triggers tab

Under the Triggers tab you specify when the job will be run.

Fields & buttons Description
Run manually The job will only be run if a user explicitly runs it from the Jobs list view or the Job edit view.
Run once at The job will be run only once at a specific date and time.
Run multiple times The job will be run repeatedly at the interval specified in the Recurrence pattern box. The Range box allows you to specify the earliest start time of the job through the Start fields, and optionally when the job should no longer be run through the End by fields.
Recurrence pattern
  • Hourly – select to run the job for example every second hour or 4 times each hour that is every 15 minutes.
  • Daily - select to run the job for example every second day or to run every weekday.
  • Weekly – Select specific weekdays to run the job on.
  • Monthly – Select date for the job to be run at either every month or for an example every second month. The job can also be run for example the third Thursday every month.
  • Yearly – Select a date for the job to be run at every year. The job can also be run for an example the third Thursday of June.
Range Select the start and end date and time for the recurrence pattern to be valid.
