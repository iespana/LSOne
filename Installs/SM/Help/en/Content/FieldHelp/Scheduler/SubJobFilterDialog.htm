Subjob From-Table Filter Dialog

General setup > Replication > Subjobs > Edit a subjob > From-table filters tab > Add/edit a from-table filter

What is the dialog used for?

  • To add/edit a from-table filter on a subjob.
  • From-table filters are used to filter the data being replicated when the subjob is run. For and example the Clean actions job in the default replication data package uses a from-table filter to delete all actions which are older than 60 days.
Fields & buttons Description
Field Select the field you want to filter by from a list of all table fields the subjob is for.
Filter type

Displays which type of filtering is being used:

  • Less - Only lines with a less value stated in field Value 1 are replicated
  • Equal - Only lines equal to value stated in field Value 1 are replicated
  • Higer - Only lines higher than the value stated in the field Value 1 are replicated.
  • Range - Only lines in between the values in fields Value 1 and Value 2 are replicated.
  • Like - Like filters the same way as the SQL Like statement. Lines that fulfill the like statement are replicated.
  • Filter - Any SQL where statement to filter which lines are replicated
Apply filter

Displays the how the filtering value is interpreted:

  • Constant - The field contains a numeric or a string value.
  • Date formula - Less/Equal/Higher and Range filtering by date. Ex. D-60 means 60 days ago and D0 is today.
  • From location - Lines only replicated if the selected field is equal to the From location (only if Filter type = Equal).
  • To location - Lines only replicated if the selected field is equal to the To location (only if Filter type = Equal).
Value 1 Enter/edit the filtering value.
Value 2 Enter/edit the to filtering value in the Range (only if Filter type = Range).
