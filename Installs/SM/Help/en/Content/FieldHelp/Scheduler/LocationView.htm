Distribution Location View

General setup > Replication > Distribution location > Edit a location

What is the view used for?

  • To view/edit the setting for the distribution location such as which data director it communicates with and which database the location uses.
Fields & buttons Description
Type

Displays the type of the distribution location:

  • Store
  • Terminal
  • General
Name Displays the description of the store, terminal or general location.
ID of external object Displays the ID of the store or terminal locations.
Enable for replication Select to enable data replication to and from this location.
Replication tab
Fields & buttons Description
Distribution server

Enter the network host name of the Data Director that handles this location.

Note: Do not use location specific names such as localhost or 127.0.0.1.

Port The TCP/IP port to use when communicating with the Data Director. Only change this field if the Data Director has been set up to use other ports than the default.
Network mode

The network mode to use when communicating with the Data Director.

  • TCP - Unsecure TCP/IP connection. This is the default recommended network mode. Use this on a local network when the machines running the Scheduler and the Data Director are not on the same Windows domain.
  • TCPS - Secure TCP/IP connection. Can only be used if the machines communicating are on the same Windows domain or the same Windows workgroup.
  • OFF - Offline replication is used to send jobs to a location with dynamic address or store that is not online all the time. The Data Director at the source location will not send the job to the destination, instead the job will just wait, with status None. The destination host will send pull request to the source Data Director asking for any jobs waiting for it, and pull them from the source.
  • MAN - Manual replication allows the Data Director to transfer job from one host to another without network connection. The source Data Director will export the job data to special output folder for the destination host, and the destination host can import this job data into the destination Data Director and be processed there.
Use database server Place a check mark in this value if the Data Director at this location should connect to a database. When unchecked, the Data Director for this location is only used to forward data packs to other Data Directors.
Database driver type

The type of database driver to use for this location:

  • MS SQL - Microsoft SQL Server with SQL Server authentication. When this is selected you must provide a user ID and a password (see below)
  • MS SQL (Windows authentication) - Microsoft SQL Server with Windows authentication. When this is selected, the Windows user account running the Data Director service must have the proper security privileges on the database server.
  • NAV Native 6.01 - Microsoft Dynamics NAV version 6.01 native database.
  • NAV SQL 6.01 - Microsoft Dynamics NAV version 6.01 using Microsoft SQL Server database.
  • OLE DB - Any OLE DB compatible database server. The default provider specified in the resulting connection string is for MS SQL Server but can be replaced by manually editing the connection string.
  • OLE DB (Windows authentication) - Any OLE DB compatible database server using Windows authentication. The default provider specified in the resulting connection string is for MS SQL Server but can be replaced by manually editing the connection string.
Company Only for NAV driver types: The company to connect to in the NAV database.
User Id The user ID to use when connecting to the database. Not required when using driver type MS SQL (Windows authentication).
Password Password for the user ID. Not required when using driver type MS SQL (Windows authentication).
Database server

The network host name of the database server.

Note: If you are using Microsoft SQL Server with a named instance, the instance name must be added to the host name, for example: MyHost\SQLExpress
Database or path The name of the database or the file path if using a file based database.
SQL Server connection type

There are 4 options:

  • (default) -
  • Named pipes - A named pipe is a named, one-way or duplex pipe for communication between the pipe server and one or more pipe clients. Named pipes can be used to provide communication between processes on the same computer or between processes on different computers across a network.
  • Shared memory - The shared memory protocol can only be used when the Data Director is running on the same machine as the SQL Server.
  • TCP/IP - The Data Director used the TCP/IP protocol to connect to the SQL server.
Database string

The connection string used by the Data Director to connect to the database. You can automatically generate this value by clicking on the Set connection string button.

Database design The database design that describes the tables and fields in the database for this location. In most setups, each location has the same database design. To create a new database design, click the Read Design button on the right hand side of the view.

Fields & buttons Description
Name

Displays the description of the terminal, store or general location. The type is identified with an icon:

  • Store
  • Terminal
  • General
ID Displays the ID of the location
Add Select this button to add members/parents to the highlighted group in the
Delete Select this button to delete the selected member/parent from the group.

Actions

  • Test connection - The Test Connection button enables you to test if the connection to the Data Director at this location is valid. If the location has a database server, the connection to the database will be tested as well.
  • Read design - The Read Design button enables you to connect to the database on this location and read design information on all tables and fields in the database. This design is then automatically assigned to the current location. You can also re-use designs amongst other locations by selecting a design from the Database design drop-down combo box.
