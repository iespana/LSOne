Centrally Suspended Transactions

Suspended transactions is a way to delay the payment, or if the customer wants to pay the items on another register or store.

Suspended transaction is created in the LS POS. When suspending a transaction on the LS POS, a transaction is sent to the LS Site Service and saved to the database, if no connection is available to the LS Site Service, the transaction is saved locally.

Information about how a suspended transaction works is in chapter Suspended Transactions.

The Site Service has a function which enables the operations Suspend and Recall transaction to work centrally.

For central suspend and recall the active Site Service profile should have the field Use central suspensions selected.

The field User confirmation:

  • It can only be selected when the Use central suspensions field is selected.
  • When selected, the POS asks for the user’s confirmation when the operation is performed.

Another feature of the central suspensions via the Site Service is that it is possible to transfer transactions to other terminals which were saved locally.

Transferring a suspended transaction:

  • Click on the Recall Transaction Button in LS POS.
  • To recall a transaction, choose a transaction and press the select button.
  • To transfer a transaction, choose a transaction and press the transfer button.

  • The Location column displays if the transaction was suspended locally or centrally.
    • Locally

    • Centrally
