Trade Agreement Discount Configurations

Trade agreement discounts can be activated and deactivated globally in the system. If discounts have been deactivated, the user can still configure them but the LS POS will not include them in its calculations.

Part of the Administrative options is a tool to configure which discount types are active in the system and include some settings for the discount calculation rules.

To open Administrative options select General setup > System > Administrative tools, select tab Active price/discount

or use the ribbon: Tools > Administration Options, select tab Active price/discount:

Administrative options tab Active price/discount:

The screen above shows how the setup is in the Demo Data company. It is very important for the person responsible for the data setup to understand how this configuration works.

Activation of Trade Agreement Discounts

These are the settings in Administrative options, Activate price/discount, to configure which trade agreements discount combinations are active for the LS POS:

Note:

  • The settings are grouped into three groups: Line discounts, Multiline discounts and Total discount.
  • The fields have check box fields, which when selected, the option is active and when clear, the option is inactive. An inactive option means that the LS POS does not include discount rules controlled by this field.
  • Each group has one or more rows and three columns to be able to control all possible combinations. When the trade agreements rules are set up then each rule identifies if it works for a single entry, group or all.

The following example explains the purpose of discounts activation:

A line discount, 10% discount to all customers, has been setup for item Punch.

  • When field Item – All customers in part Line discounts is inactive, item Punch receives no discount when sold in the LS POS.

Site Manager view:

POS sales line view:

When field Item – All customers in part Line discounts is active, the item Punch receives a 10% discount in the LS POS.

Note: The LS POS has to be restarted for these changes to take effect.

Calculation Settings

The system has some configurations for the calculation settings, these control how the LS POS handles line and multiline discounts and when customer and periodic discounts are calculated in the sale.

Discounts:

Line: The line discount is selected over the multiline discount.

Multiline: The multiline discount is selected over the line discount.

MAX (Line, Multiline): The discount with the higher discount % is selected.

MIN (Line, Multiline): The discount with the lower discount % is selected.

Line + Multiline: The discount % is added together and added to the item.

Line * Multiline: The discount % is multiplied and added to the item.

Customer discounts:

These apply to trade agreement discounts.

Always calculate: The LS POS calculates trade agreement discounts every time an item is inserted or changed.

Calculate on total: The LS POS calculates trade agreement discounts only when a payment is processed (that is when the total mode is reached). Operation Display Total calculates the totals when used no matter what the configuration is in this field.

Periodic discounts:

These apply to periodic discounts which are discount, multibuy and mix and match offers.

Always calculate:

The LS POS calculates periodic discounts every time an item is inserted or changed.

Calculate on total: The LS POS calculates periodic discounts only when a payment is processed (that is when the total mode is reached).
