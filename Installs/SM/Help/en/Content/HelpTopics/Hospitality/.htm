4.1.2 General Tab

The general tab has configurations regarding the look and feel of the main Table view where the cashier can select a table and work with the transaction on it.

Overview

Only Button format is implemented in the current version of the LS One Hospitality.

Staff takeover in transaction

The options are not implemented in the current release. In this version simply select button Change staff in Table View, select the table and confirm that you want to overwrite the staff member on this transaction by selecting Yes.

Touch buttons

Each Hospitality type can have its own Touch button layout. If it is not set, the POS selects the layout depending on Store, Terminal and Staff layout configuration.

Top POS menu ID

The look of the button the user clicks to select the current Hospitality type.

POS Logon Menu ID

The button menu on the right side of the Table view.

Stay in POS after transaction

If not selected, the POS will automatically return to the Table view after the transaction is finished. If it is selected, the user will always have to select the Exit Hospitality POS operation in the POS to return to the Table view.

Default type

Select this check box to make this hospitality type the default type, used when the table view is skipped. See chapter Functionality Profile in Hospitality setup Functionality Profile in Hospitality setup for more information on Skip table view.

 

Send to station

Controls when the POS asks the user if items, that have yet to be printed at Restaurant stations, should be printed. See chapter Station Printing Station Printing for more information.

  • None: No station printing done
  • At POS Exit: Station printing only done when Exit Hospitality POS operation is run
  • At POS Posting: Station printing only done when transaction payment is concluded
  • At POS exit & POS posting: as the two above

 

4.1.3 Table button/Graphical layout

This tab configures how the table layout in the Table view should look and how many tables are available for each hospitality type.

Dine-in table required:

Not implemented in the current release.

Dining table layout id:

The layout for the Table View. See more information in section 4.1.5 Dining Table Layout View. Click on the Edit or Add button (see chapter 4.1.4 New Restaurant Layout) to edit or add a layout.

Table button description:

Controls the descriptions on the tables. The options are:

  • Number
  • Description
  • Number + Description
  • Description + Number

Table button staff description:

Controls which staff description is added to the table button:

  • Staff id
  • Receipt name

 

 

4.1.4 New Restaurant Layout

General setup > Hospitality > Hospitality types > [select hospitality type] > Table button/graphical layout tab > Dining table layout id Add button

 

 

 

Hospitality type

Displays the hospitality type in which the dining table layout is being created.

Description

The default description which will be displayed on all tables depending on the Table button description configuration of the Hospitality type. Each table can then be given a specific description if needed in the Edit restaurant dining table dialog.

Dining table rows

Indicates how many rows of tables are available and visible in the Table view on the POS.

Dining table columns

Indicates how many columns of tables are available and visible in the Table view on the POS.

Copy from

If the user would like to use an existing dining table layout it can be selected here and its configuration will be used to create the new layout.
