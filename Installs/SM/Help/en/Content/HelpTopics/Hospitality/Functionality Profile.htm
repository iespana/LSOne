Functionality Profile in Hospitality setup

Sites > Stores > [select store] > Settings tab > Functionality profile > Edit button > [select profile] > Hospitality tab

To activate the Hospitality functionality on the terminal, the Functionality profiles that the store (or terminal) are using need to have the Is hospitality profile checkbox selected.

This is done so that a restaurant can have one terminal that is a hospitality terminal and another that is a normal store terminal (for gifts perhaps).

The functionality profile can either be on the restaurant or the terminal, when LS One POS is started and the terminal has a functionality profile that is used. Otherwise the store functionality profile is used.

If no functionality profile is selected on the store, either select a new one or go to the terminal functionality profile.

 

Store View, Settings Tab

Functionality Profile View, Hospitality Tab

Skip table view: If this check box is selected, the POS will not show a table view, but go directly to the default hospitality type view. Select the default type in Hospitality Type. This can be useful if you have only one POS and the customer pays when they order.

 

Item changes after split bill: If this check box is selected, the POS will allow changes to a split item, like set qty, void item, and so on.
