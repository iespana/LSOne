Assign Infocodes to Suspended Transaction Types

On the POS, the operation Suspend can be used to suspend a transaction that is in the process of making a sale. Sometimes it might be easier to store the transaction than to redo it later.

The suspended transaction can be linked to a certain transaction type. One feature of the suspended transaction is that you can assign an infocode to it. This can, for example, assign a reason for the transaction’s suspension.

To assign an infocode to suspension transaction type:

  • Select Retail > Transactions > Suspension transaction type.
  • Select and edit the suspended type that is going to have the infocode assigned to it.
  • Press the add button to add a label to the suspension type.
  • In the Suspension field dialog select the type Infocode and select which infocode should be called when a transaction is suspended on this suspension type.
  • Press OK to add the infocode.
