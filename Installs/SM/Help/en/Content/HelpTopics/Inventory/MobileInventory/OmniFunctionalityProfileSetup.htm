Omni Functionality Profile Setup

For each mobile device connected to LS One you need to create a terminal in the store the mobile device should be connected to and this terminal needs a functionality profile with omni specific settings.

These settings include which printer station the LS Omni Mobile POS should connect to and which is the Startup menu of the LS Omni Mobile Inventory app.

An example terminal and functionality profile are imported with the Omni specific Omni data.xml default data pack. See here how to import default data packs.

 

Follow these steps to create the omni functionality profile:

  1. Go to Sites > Profiles > Functionality and press the add button .
  2. Give the new functionality profile a description.
  3. Press OK and the Functionality profile view is opened
  4. Go to the Omni tab.
    5. For the LS Omni Mobile POS you need the following settings:

    Suspension type: This should be a suspension type with out any suspension questions

    Printing station: if you want to print the receipts you have to select which printing station to connect to.

    Item image lookup group: Select a special group of items that will populate the item image lookup list in the mobile POS.

     

    For the LS Omni Mobile Inventory app you need the following setting:

    Main menu: Select which of the Omni button menus opens as the startup menu.

  5. When you have selected each of these settings save the profile and add it to your omni terminal.
