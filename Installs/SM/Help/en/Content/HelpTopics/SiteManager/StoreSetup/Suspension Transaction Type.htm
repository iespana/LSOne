Suspension Transaction Type

Transactions can be suspended in the LS POS with the operation Suspend transaction.

The default data has the operation Suspend transaction on three buttons: Suspend, Delivery and Modifications. The difference for these buttons, apart from the description is that they have a different value for their operation parameter.

The suspension type has to be defined with the operation Suspend transaction. This is also true of the operation Recall Transaction.

The button Suspend in the Demo Data has the suspension type Default:

Suspension types have two main purposes:

  • With them it is possible to set up extra information which needs to be added when transactions are suspended in the LS POS.
  • It is additional grouping for suspended transactions. The suspension type is registered with the suspended transactions and is therefore one of the filter fields in Site Manager’s suspended transactions.

Suspension transaction types are created and edited in Sales> Suspended > Suspension transaction types:

The Demo Data has three suspended types set up:

The Suspended type card:

It is possible to add in one or more labels for suspension types which can be of a different field type. The labels act in the LS POS by requesting some extra information with the suspended operation. The types can be Text, Customer, Name, Address, Infocode or Date. When the suspension type is active for a transaction being suspended, the system automatically displays windows to gather information according to the suspension type labels. The setup here decides the behavior for the suspended operation in the LS POS.
