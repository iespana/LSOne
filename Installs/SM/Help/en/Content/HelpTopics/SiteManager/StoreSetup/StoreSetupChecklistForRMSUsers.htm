Store setup checklist for RMS migration customers

Consider carefully the following information before importing data from RMS

  • All users that are imported will have their passwords reset to 1234 including users that already exist in the LS One database and have the same login as the RMS users. This includes the "admin" user
  • User permissions will need to be configured specifically on each user after the import as well as other settings that need to be set on each user
  • Currency denominations need to be created by customer or partner
  • All retail items are imported as standard retail items
  • Imported vendors all have default tax calculations method set as NO TAX
  • Information regarding customers' credit limit and their current balance is not imported
  • Terminals need to be configured for each store by customer or partner
  • Discounts and vouchers are NOT imported
  • Prices are imported either with or without tax depending on the tax configurations in RMS. To calculate the other value you need to configure the Default tax store to have price settings as either Price or Price with tax. When Recalculate prices is clicked the other value is calculated i.e. Default tax store is configured to be Price with tax then the Price (without tax) is calculated from that value using the tax group on the store using the tax group that is set on the default tax store

After importing it is crucial to verify that all tax group configurations are correct, and that sales tax codes are set on appropriate tax groups

It is important to note that the information that is imported from RMS to LS One does not satisfy requirements made by tax authorities for storing sales and tax information
  • Please note that when transactions are imported, the RMS migration tool is importing as much of the transactions as is possible. The difference in information between the two systems is substantial and the RMS migration tool is importing as much as it can of the information available
  • Any sales information that are migrated between the systems should only be used for reporting purposes, comparing sales information between years, forecasting and etc.
  • The LS One system will not be able to use these transactions for returns or to printout copies of previous receipts for example. To view these transactions any reports will have to display sales transactions with EntryStatus = 2 (as well as 0 which is a normal sale transaction)

 

This chapter includes a list of tasks that need to be performed in Site manager, before and after migration of data from RMS to LS One

Before migration of data
Setup
1. Create and select company currency
  All the currencies that are going to be used by the currency payment type are set up here. Create an exchange rate for the new currency. All prices set in the Site Manager will be in this currency unless otherwise stated.
2. Insert company information
  Company description, address, contact information and logo.
3. Create a price group
  Enter a description for a price group and determine whether the price includes tax.
Tax
4. Create a sales tax code
  Sales tax codes include tax percentage for the sales tax calculation. The sales tax codes are then used in the setup of item sales tax group and the customer and store sales tax group.
5. Create a customer and sales tax group
  Create a customer and sales tax group and add sales tax codes to the group.
6. Create an item sales tax group
  Create item sales tax group and add sales tax codes to the group.
7. Select default tax calculation store
  Set default store for Site manager in order to get all of the default data for currency.
Bar codes
8. Create bar codes
  Setup bar codes. Each retail item can have anything from no bar codes up to many bar codes.
Profiles
9. Create a visual profile
  Settings regarding screen resolution, type of terminal, whether cursor should be hidden or not, whether currency symbol should be shown in columns or not and identifying screens.
10. Create a functionality profile
  Various settings regarding terminals, retail items, staff, start of day, hospitality, X/Z Report and when selected infocodes should be triggered.
11. Create a hardware profile
  Hardware profiles are set on terminals to connect to hardware such as Line display and Printers.
12. Create Site service profile
  Setup and test connections.
Touch Buttons
13. Create POS button grid menus
  Create different menus with this operation. Menus are either attached to button grids used in touch button layouts or to menu buttons that are part of the menus.
14. Create a touch-button layout
  Customize the POS layout. It is possible to have a different layout for each store, terminal or user. To make the setup easier it's possible to insert the default demo touch button layout and then customize it as needed.
New Store
15. Create new store
  Create a new store in the system. Add terminals and allowed payment types to the store after importing.
16. Settings
  Store configurations and default values. Select a sales tax group for the store.
17. Price groups
  Select a price group to link to a store.
18. Price/tax calculations
  Configure price and tax calculation for a store.
After migration is finished

Some of the tasks may not be needed in every setup and are optional for the system; an example of this is the setup of hospitality types

Italics highlighted: Specific to your store settings, not necessary for general setup.

Note: This list is designed to aid you in your general setup, and does not necessarily include  all requirements specific to your setup

Store configuration
1. Allowed payment types
  Add payment types to a store. Each store should have at least one payment type set up and it is quite normal to have few of them. Payment types are set up separately for each store.
2. Statement settings
  Configure the handling of statements which are then operated in the Site Manager for transactions created in the POS.
3. Price/tax calculations
  Configure price and tax calculation for a store.
4. Select Site service profile for the store
  Choose the site service profile for the store
5. Configure settings on terminals
  Decide for example if terminal should be included in End of day statements.
6. Income/Expense accounts
  Income and expense accounts for a store are set up here.
7. Hospitality types
  Configure which hospitality types should be available in a restaurant and add them here. Hospitality types are areas in the restaurant such as “bar” or “patio.” They can have different sales type settings (tax group and price group), table layouts, and so on.
8. Menu types
  For the hospitality system. Menu types are related to printing or sending orders to the kitchen.
9. Tender declaration
  View a list of the tender declarations that have been made in a store. A tender declaration can be made on the LS One POS terminal.
10. Suspended transaction
  Field which determines if a suspended transaction can exist when end of day operations are processed.
Forms, units, users, infocodes, currency and retail group/department setup
1. Create a receipt layout and other forms that you need
  Create i.e. a customer receipt layout, gift receipt, invoice and other forms needed. Form types come with the installation of the Site manager. Demo receipt layouts can be inserted and updated to your needs.
2. Create units
  Create scale units for Gram, Kilo gram, Ounce and Pound. Unit descriptions and number of minimum/maximum decimals allowed for the item in sale.
3. Create unit conversions
  Unit conversions can be set up globally or for specific items.
4. Create retail groups
  Create a retail group.
5. Configure users permissions
  Each user's permissions is needed to be configured specifically since all imported users are enabled and have same permissions.
6. Add button properties to the touch button layout
  Configure a button in a menu. There are over 100 operations in the system. You can set up the POS layouts differently, changing the menus and operations in them.
7. Review and update if necessary default functions for payment types
  Set default functions and limitation on payment types.
8. Review and update customer's settings
  Manage customer's settings such as address, discounts, credit limit and blocking status.
9. Setup card types
  Create card types that can be used as payments
10. Create a retail department
  Create a retail department. Every retail item belongs to one retail group and each retail group can belong to a retail department.
11. Configure terminals
  Configure terminals to have them included in a End of day statement and allow posting.
12. Create a special group
  Create a special group. An item can belong to many Special Groups
13. Create infocodes
  Infocodes are used to register additional information about transactions made in the POS. Infocodes are set up in the Site Manager and then assigned to different actions. The setup of infocodes is quite flexible.
Administration options setup
1. Choose Administrative options
  Different things for the company and stores are set up and configured. The setup in Administrative options should be one of the first tasks before the data for the company is inserted. 
2. Localization
  Decide in which order names are displayed/printed and what type of address fields should be used.
3. Security
  How to control login for users, name of active directory (domain), enable or disable write auditing.
4. Active price/discount
  Decide which type of price calculation and discount calculation combination rules can be used in the LS POS.
5. Discount settings
  Set calculation settings for an item and a customer.
6. Scale units
  To fulfill the Scale Certification requirements, four scale units have to be selected from the units list for the scale to work properly.
7. Number sequences
  View, add or edit a number sequence. Number that sequences come with the system are viewable here.
8. Store management
  Select the store which decides the default tax calculation. All prices in the system will be shown with this store’s tax settings. NOTE: Prices need to be recalculated and updated after the import so that prices will be calculated with tax rather than without. Click the Recalculate button to recalculate the prices. Decide where the user is allowed to enter ID's manually i.e. Item ID, Customer ID etc.
9. Inventory settings
  Manage the number of items that are allowed to be higher than the ordered quantity when receiving items.
10. Site service
  Setup and test connections.
11. Decimal formats
  With this configuration it is possible to set up minimum and maximum numbers of decimals for numbers in the system.
12. Cash denomination
  Cash denomination need to be setup for currency. Select amount that is accepted by the store in each currency.
13. Customer ledger
  Deletes all customer ledger entries and recreates them.
14. Blank operations
  Partners can use blank operations to add functionality and operations that are missing from the LS POS.
