Setting the scheduler service

In order to communicate with the Scheduler service, the Scheduler plugin in the Site Manager must be told where that service is hosted. To do this go to General setup > Replication > Settings. The Settings view will be shown.

The Host field contains the name of the machine hosting the Scheduler service. By default it is set to the current machine. If the Scheduler service is not set to run on this machine, change this field to the machine that the scheduler service is running on (this is the same machine that was used to install the Data Director).

The Network mode controls the method of communicating with the Scheduler service. It defaults to TCP. If you require secure communications and have the Store Controller machine and the Scheduler service machine are on the same Windows domain you can change this to TCPS.

The Port identifies the service on the host machine. Unless the port of the Scheduling service has been explicitly changed, you should leave this field empty showing the default value in dimmed text.

For further information on network mode and ports, see the description of distribution locations in section Using the Scheduler.
